# 전체 점검 버그 리포트

## 왜 아무것도 안 되는가

**근본 원인: imageManager.js의 `updateList()`에서 변수 순서 오류**

```js
// 라인 192: hasIssue를 사용
const hasErr = hasIssue;  // ← 아직 선언 안 됨!

// ... 중간에 다른 코드 ...

// 라인 209: hasIssue를 선언
const hasIssue = hasWarning || hasMulti;  // ← 여기서 선언
```

`const`는 선언 전에 접근하면 `ReferenceError` 발생 (Temporal Dead Zone).
→ `updateList()` 크래시
→ 이후 `select(0)` 실행 안 됨
→ 캔버스 표시 안 됨
→ **아무것도 안 됨**

## 발생 경위

중복값 뱃지 기능을 추가하면서 `hasIssue` 변수를 만들었는데,
뱃지 표시 코드(라인 209 부근)는 밑에 있고,
CSS 클래스 적용 코드(라인 192)는 위에 있어서 순서가 뒤집힘.

## 전체 버그 목록

| # | 심각도 | 파일 | 내용 |
|---|--------|------|------|
| 1 | **치명** | imageManager.js:192 | `hasIssue` 선언 전 사용 → updateList 크래시 → 전체 불능 |

## 수정 방안

1. `hasWarning`, `hasMulti`, `hasIssue` 계산을 `hasErr` 사용 전으로 이동
2. ROI 추가 시 `btnAnalyze.disabled = false` 추가
3. template.js에서 choiceLabels deep copy
