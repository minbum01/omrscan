## 2. 미구현 기능 상세 설계

### 2.1 세션 저장/불러오기

**목적**: 작업 중인 상태를 파일로 저장하여 나중에 이어서 작업

**저장 형식**: JSON 파일 (.omr-session)
```json
{
  "version": "1.0",
  "savedAt": "2026-03-18T...",
  "answerKey": { ... },
  "images": [
    {
      "name": "파일명.jpg",
      "dataUrl": "data:image/jpeg;base64,...",
      "rois": [ { "x":10, "y":20, "w":300, "h":500, "settings": { ... } } ],
      "results": [ ... ],
      "gradeResult": { ... }
    }
  ]
}
```

**UI 위치**: 헤더에 "세션 저장" / "세션 불러오기" 버튼
- 저장: Blob → download (JSON 파일)
- 불러오기: file input → JSON 파싱 → 상태 복원

**주의사항**:
- 이미지를 base64로 포함하므로 파일 크기가 클 수 있음
- 저장 전 확인 토스트, 불러오기 전 현재 작업 덮어쓰기 경고

---

### 2.2 양식 저장/불러오기

**목적**: ROI 설정만 별도로 저장하여 동일한 OMR 양식에 재사용

**저장 형식**: JSON 파일 (.omr-template)
```json
{
  "version": "1.0",
  "name": "5과목 공무원 답안지",
  "rois": [
    {
      "x": 50, "y": 100, "w": 200, "h": 400,
      "settings": {
        "startNum": 1,
        "numQuestions": 20,
        "numChoices": 5,
        "orientation": "vertical"
      }
    }
  ]
}
```

**UI 위치**: 우측 패널 "영역 설정" 단계 하단에 "양식 저장" / "양식 불러오기" 버튼
- 양식 불러오기 → 현재 이미지에 ROI 자동 적용

**사용 시나리오**:
1. 첫 번째 답안지에서 ROI를 설정
2. "양식 저장"으로 템플릿 파일 다운로드
3. 다음에 같은 양식의 답안지를 채점할 때 "양식 불러오기"로 즉시 적용

---

### 2.3 ROI 방향 설정 (가로/세로)

**목적**: OMR 마킹의 배치 방향을 지정하여 정확한 행/열 분석

**두 가지 방향**:
- **세로 (vertical)**: 문항이 위→아래로 나열, 선택지가 좌→우 (일반적인 OMR)
  ```
  Q1: ①②③④⑤
  Q2: ①②③④⑤
  Q3: ①②③④⑤
  ```
- **가로 (horizontal)**: 문항이 좌→우로 나열, 선택지가 위→아래
  ```
  Q1 Q2 Q3
  ①  ①  ①
  ②  ②  ②
  ③  ③  ③
  ```

**UI 위치**: 우측 패널 ROI 설정 폼에 "방향" 드롭다운 추가
- 기본값: 세로 (vertical)

**엔진 변경**: omrEngine.js의 analyzeStructure()에서 방향에 따라 행/열 그룹핑 축을 전환
- 세로: Y축 기준 행 그룹핑 → X축 정렬 (현재 로직)
- 가로: X축 기준 열 그룹핑 → Y축 정렬 (축 반전)

---

### 2.4 이미지 개별 삭제

**UI**: 이미지 목록 각 항목에 ✗ 삭제 버튼 (hover 시 표시)
- 클릭 → 토스트로 "삭제됨" 알림
- 현재 선택된 이미지 삭제 시 → 다음 이미지로 자동 전환
- 마지막 이미지 삭제 시 → 빈 상태로 복귀

---

### 2.5 ROI 개별 삭제

**방법 1**: 우측 패널 ROI 설정 카드에 ✗ 삭제 버튼 추가
**방법 2**: 캔버스에서 ROI 클릭 → 선택 → Delete 키로 삭제

**구현 방식**: 방법 1 우선 (더 직관적)
- 각 ROI 설정 카드 우상단에 삭제 버튼
- 삭제 시 분석 결과 초기화 + 캔버스 리렌더링

---

### 2.6 팬 모드 수정

**현재**: 이동 모드(V) 선택해도 실제 드래그 이동 안 됨
**수정**: canvasContainer의 scrollLeft/scrollTop을 마우스 드래그로 제어

## 3. 수정된 헤더 버튼 구성

```
[테스트양식] [이미지업로드] [정답입력] [일괄채점] | [양식저장] [양식불러오기] | [세션저장] [세션불러오기] | [CSV] [Excel]
```

→ 버튼이 너무 많으므로 그룹화:
```
[테스트양식] [이미지업로드]  |  [정답입력] [일괄채점]  |  저장▼ (드롭다운: 세션저장/양식저장/CSV/Excel)  |  불러오기▼ (드롭다운: 세션불러오기/양식불러오기)
```
