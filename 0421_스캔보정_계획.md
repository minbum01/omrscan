# OMR 스캔 보정 계획

## 문제 현황
스캔별로 기울어짐, 위치 밀림, 마킹 오류, 문항 누락 등 다양한 문제 발생

---

## 1. 자동 기울기 보정 (Deskew)
**문제**: 스캐너에 비뚤게 들어가서 이미지 전체가 1~5도 기울어짐 → 모든 감지 실패  
**영향**: 가장 큼 — 기울어지면 행/열 정렬, 블롭 감지 전부 무너짐

**해결 방향**:
- Hough Transform으로 직선 검출 → 기울기 각도 계산
- 또는 OMR 외곽 직선/검정 마커의 각도 분석
- 계산된 각도만큼 canvas rotate로 자동 보정
- 일괄 처리 시 각 이미지별 자동 적용

**적용 시점**: 분석 전 전처리 단계 (Stage0)

---

## 2. 기준점(앵커) 기반 위치 보정
**문제**: 같은 양식인데 이미지마다 스캔 위치가 좌우/상하로 밀림  
**영향**: ROI 좌표가 정확해도 실제 버블 위치가 다름

**해결 방향**:
- OMR 용지의 고정 마커(검정 사각형, 코너 마크, 타이밍 마크) 자동 감지
- 양식 기준점 대비 현재 이미지의 offset 계산
- ROI 좌표를 offset만큼 자동 보정
- 고정 마커 없으면: blobPattern의 열 평균 위치를 기준으로 상대 offset 계산

**적용 시점**: 양식 적용 후, 분석 전

---

## 3. 버블 인식 오류 대응
**문제**: 연하게 칠한 마킹, 지우개로 지운 흔적, 마킹 번짐  
**현재 대응**: composite score (f×0.4 + e×0.35 + qMin×0.25) + 1.5x 링 검증

**추가 방향**:
- **이미지별 최적 진하기 자동 탐색**: 분석 전에 여러 intensity로 시도 → 가장 안정적인 결과 채택
- **적응형 임계값**: 이미지 전체 밝기/대비에 따라 마킹 판별 임계값 자동 조정
- **지우개 흔적 필터**: erodedFill이 낮고 centerFill만 높은 경우 → 지운 흔적으로 판별

---

## 4. 특정 문항 누락
**문제**: 일부 행/열이 감지되지 않음  
**현재 대응**: Stage4 누락 행 복원 (gap median 기반)

**추가 방향**:
- **양식 패턴 강제 적용**: blobPattern이 있으면 모든 행/열 위치를 패턴 기반으로 강제 생성
- **기대 행 수 vs 감지 행 수 차이 분석**: numQuestions 하드코딩 값과 비교하여 정확한 위치 추정
- **열 평균 기반 보간 정확도 향상**: 정상 행의 열 위치 패턴으로 누락 행의 블롭 위치 정밀 추정

---

## 우선순위
1. **자동 기울기 보정** — 임팩트 최대, 다른 모든 보정의 전제조건
2. **기준점 기반 위치 보정** — 양식 활용도 향상
3. **진하기 자동 탐색** — 인식률 향상
4. **양식 패턴 강제 적용** — 누락 방지

## 구현 원칙
- 기존 Stage1~5 로직 변경 없음
- 전처리 단계(Stage0)로 추가
- 사용자 선택적 활성화 (토글)
- 일괄 처리 시 자동 적용
