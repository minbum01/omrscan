# 양식 저장 시 블롭 "상대 규격" 저장 → 재사용 시 감지 힌트로 활용

## 핵심 개념

**양식 저장 시 블롭의 절대 좌표가 아닌, 블롭들 간의 "상대적 거리/크기"만 저장**해서
새 OMR을 판독할 때 **BFS 보조 지표**로 사용한다.

> "1번과 2번 버블은 이만큼 떨어져 있구나, 세로박스일 땐 이렇구나, 가로박스일 땐 이렇구나"
> — 이 정보를 다음 OMR 판독에 힌트로 쓴다.

좌표 하드코딩이 아니라, **규칙성(regularity)의 기준값**을 보존하는 것.

## 저장할 데이터 (절대 좌표 아님)

```json
{
  "intensity": 115,                  // 양식 제작 당시 진하기
  "rois": [
    {
      "x": 100, "y": 200, "w": 300, "h": 400,
      "settings": { ... },
      "blobPattern": {
        "numRows": 20,
        "numCols": 5,
        "orientation": "vertical",

        "bubbleWRatio":    0.05,     // 버블 폭 = ROI 폭 × 0.05
        "bubbleHRatio":    0.04,     // 버블 높이 = ROI 높이 × 0.04
        "colSpacingRatio": 0.15,     // 인접 열 간 거리 = ROI 폭 × 0.15
        "rowSpacingRatio": 0.048,    // 인접 행 간 거리 = ROI 높이 × 0.048
        "sampleBlobCount": 100       // 규격 계산에 쓴 블롭 수 (신뢰도)
      }
    }
  ]
}
```

**주의: 어떤 블롭이 어디 있는지(cx, cy)는 저장하지 않는다.**
저장되는 건 "간격"과 "크기"뿐. 새 OMR의 ROI 박스 크기에 맞춰 비율로 스케일링되어 적용됨.

## 동작 흐름 (새 OMR 분석 시)

### 1. 양식 로드 → blobPattern 파싱

```
저장된 비율을 현재 ROI 크기에 맞춰 실제 픽셀 값으로 환산:

expectedBubbleW = ROI.w × bubbleWRatio
expectedBubbleH = ROI.h × bubbleHRatio
expectedColGap  = ROI.w × colSpacingRatio
expectedRowGap  = ROI.h × rowSpacingRatio
```

### 2. BFS 기본 감지 (기존)

Stage 1에서 BFS로 블롭 후보 찾음. 기대값과 비교해서:
- BFS가 찾은 블롭 폭/높이가 `expectedBubbleW/H`와 근접 → 정상
- 너무 작거나 큰 블롭 → 필터링 강화

### 3. 감지 결과와 규격 비교

```
BFS 결과: 행 20개, 열 5개 기대 → 100블롭 예상

Case A: 100블롭 모두 감지, 간격도 정상 → 그대로 사용
Case B: 95블롭만 감지 (5개 누락)
  → 누락된 위치를 주변 블롭 + 저장된 간격으로 보간
  → 예: 행 3의 열 2가 없음 → 행 3의 열 1, 3 평균 + colSpacing으로 위치 추정
Case C: 70블롭만 감지 (심각) → 규격 신뢰도 낮음, BFS 결과만 사용
```

### 4. 열/행 정렬 시 힌트 활용

`_stage3_filterByColumnConsistency`에서 `tolerance` 값을 저장된 `bubbleWRatio × ROI.w × 0.5` 같이
동적으로 설정 → 더 정확한 열 정렬.

## 핵심 이점

1. **BFS가 놓친 블롭 위치 복원** — 규격을 알면 빈 자리 좌표 계산 가능
2. **다른 OMR 크기에도 적용** — 비율 기반이라 ROI 크기 달라도 스케일링
3. **과감지 필터링** — 기대 크기와 맞지 않는 블롭(숫자 획 등) 자동 배제
4. **좌표 고정 아님** — 용지가 조금 밀리거나 회전해도 BFS가 찾은 실제 위치 기반으로 동작

## 구현 범위

### Phase 1 — 저장/로드 (완료)

- [x] `template.save()`에서 감지된 블롭들의 평균 **간격/크기 비율** 계산 → 저장
- [x] `template.load()`에서 `blobPattern` 복원 → ROI에 저장

### Phase 2 — 감지 힌트 적용 (예정)

- [ ] `analyzeROI`에 `blobPattern`이 있으면 예상 규격 계산
- [ ] Stage1 BFS 필터에서 기대 크기 기준으로 이상치 배제
- [ ] Stage1.5 또는 Stage4에서 누락 블롭 위치 보간
- [ ] 허용 오차(`tolerance`)를 `bubbleWRatio`에 맞춰 동적 조정

### Phase 3 — 고도화 (나중)

- [ ] 여러 OMR의 패턴 평균화 (더 안정적 규격)
- [ ] 패턴과 BFS 결과 불일치 감지 시 경고 (폼이 다른 OMR 경고)

## 적용 사례

### 양식 제작 시
```
사용자: 첫 OMR에 박스 그림 → Stage1~4 실행 → 블롭 정상 감지 (100/100)
양식 저장 클릭 → template.json에 저장:
  - 진하기 115%
  - ROI별 { colSpacingRatio: 0.15, rowSpacingRatio: 0.048, ... }
```

### 양식 불러와 다른 OMR 채점 시
```
사용자: 양식 불러오기 → 다른 OMR 이미지에 박스 자동 배치
        → 진하기 115% 자동 적용
        → 일괄 채점 시작

Stage1 BFS: 92블롭만 감지 (8개 누락, 마킹 약해서)
Stage2 직전: 저장된 규격으로 누락 8개 위치 계산 → 완전한 20×5 그리드 생성
Stage2: 모든 100위치에서 sampleCell → 정상 채점
```

### 기존 방식 vs 개선 방식

| | 기존 | 개선 |
|---|------|------|
| BFS가 놓친 블롭 | `undetected: true` → 누락 문항 | 규격으로 위치 보간 → 정상 감지 |
| 진하기 설정 | 매번 수동 조정 | 양식에 저장된 값 자동 적용 |
| 이상한 크기 블롭 | 종횡비만 체크 | 기대 크기 대비 편차로 추가 필터 |
| 다른 크기 ROI | 템플릿 재조정 필요 | 비율 기반이라 자동 적응 |

## 구현 가능 여부: ✅ 즉시 가능

- 저장할 정보: 집계값 4~6개뿐 (극히 경량)
- 좌표 하드코딩이 아니라 위치 흔들림/회전에 강건
- BFS 결과를 완전히 대체하지 않고 **보조만** → 안전 (패턴이 틀려도 BFS fallback)
