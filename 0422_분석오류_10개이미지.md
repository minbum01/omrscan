- 10개 선택지 중 5~6개만 BFS 감지
- 나머지 열평균이 NaN → 블롭 좌표도 NaN
- **수정 필요**: 양식 패턴의 행간격으로 NaN 위치 계산 (현재 실패)

### C. 열간격 축소/불일치 (042, 071)
- 실제 블롭 간격이 양식 기대값과 크게 다름
- 양식 절대 픽셀값이 이미지 스케일에 맞지 않음
- **수정 필요**: 양식 패턴을 비율 기반으로 적용하거나, 실제 감지 간격에 맞춰 조정

### D. 노이즈 블롭으로 열 위치 밀림 (032, 172107_005)
- 행 내에 노이즈 블롭이 끼어서 열 위치가 밀림
- 밀린 위치에서 샘플링 → comp 점수 전체적으로 낮아짐 → 답 판별 불가
- **수정 필요**: 양식 패턴의 열 간격으로 블롭 위치 검증/보정, 비정상 간격의 블롭 제거

## B 유형 근본 원인 상세 (110, 005, 036, 105)

**Stage1에서 BFS가 블롭을 못 찾은 게 아님** — 오히려 **너무 많이 찾음**.

110.jpg 행 데이터:
```
행1: 13블롭 — (13,31 14x24) (14,32 9x13) (13,92 14x23) (13,92 5x12) ...
행2: 15블롭 — (56,31 14x24) (56,32 9x13) (56,92 14x23) (55,92 4x12) ...
```

각 버블 위치에서 **본체 블롭(14×24) + 작은 노이즈 블롭(9×13, 5×12 등)이 쌍으로 감지**.
양식 기대 10개 × 7행 = 70블롭인데, 실제 102블롭 감지. 약 30개가 노이즈.

**Stage1.5 colGap 검증에서 병합:**
```
[30,92,92,152,152,213,213,274,335,336] → 병합4 → [30,92,152,213,274,335] = 6개
```

같은 위치의 블롭 쌍이 열 평균을 중복시킴 → 병합 후 6개만 남음 → **7~10번 선택지 소실**

**즉, "못 찾은 것"이 아니라 "노이즈 블롭 때문에 병합 과정에서 소실된 것"**

# 교정 기획 (v2)

## 근본 원인

4가지 유형 모두 **BFS 감지 결과의 후처리(그룹핑/필터링) 문제**로 귀결됨.
BFS 자체는 블롭을 찾긴 하지만:
- 노이즈 블롭을 같이 잡아서 그룹핑이 꼬이거나 (B, D)
- 가로형 스왑 로직 버그 (A)
- 이미지 스케일 차이 미대응 (C)

**양식 패턴은 그리드를 짜기 위한 게 아니라, BFS 결과를 검증/보정하기 위한 도구.**

---

## 교정 방향: BFS 유지 + 양식 힌트로 검증/보정

### 핵심 원칙
- **BFS는 그대로 실행** — 블롭을 찾는 1차 수단
- **양식 힌트로 BFS 결과를 후처리** — 노이즈 제거, 누락 보간, 간격 보정
- **기존 코드 수정 최소화** — 후처리 단계를 추가하는 방식

---

## A 유형 교정: 가로+길쭉 행/열 스왑 버그 (003, 021)

**원인**: `_stage1_findBubbles_elongated`에서 가로형일 때 rowGroups/colGroups 스왑 로직 결함

**수정**:
- `analyzeROI` 내 가로형 분기에서 rowGroups ↔ colGroups 스왑 코드 점검
- rowGroups가 0개일 때의 fallback 처리 추가

**위치**: `omrEngine.js` analyzeROI() 내 `rowGroups = isVert ? grid.rowGroups : grid.colGroups` 부근
**영향**: 가로형 전용 수정. 세로형 로직 변경 없음.

---

## B 유형 교정: 노이즈 블롭으로 열 소실 (110, 005, 036, 105)

**원인**: BFS가 각 버블 위치에서 본체(14×24) + 노이즈(8×13)를 쌍으로 감지
→ 열 평균에 중복 발생 → colGap 병합 시 일부 열 소실 → NaN

**수정 — Stage1 직후 노이즈 블롭 제거**:
1. 양식 힌트의 `bubbleW × bubbleH` 기준으로 **크기 필터** 강화
   - 양식 버블이 13×21이면, 면적의 50% 미만인 블롭(8×13=104 < 13×21×0.5=136.5) 제거
2. **같은 위치에 겹치는 블롭 제거** — 두 블롭의 중심 거리가 bubbleW/2 이내면 큰 것만 유지
3. 제거 후 열 그룹핑하면 중복 없이 10개 유지

**위치**: Stage1과 Stage1.5 사이에 노이즈 필터 추가 (새 코드)
**영향**: 기존 Stage1, Stage1.5 로직 변경 없음. 중간에 필터만 삽입.

---

## C 유형 교정: 간격 스케일 불일치 (042, 071)

**원인**: 양식 기준 이미지와 실제 스캔 이미지의 해상도/크기가 다름
→ 양식 간격(절대px 61)과 실제 간격(31 또는 25)이 불일치

**수정 — 양식 간격을 비율 기반으로 적용**:
1. 양식의 `colXRatios` (비율)을 현재 ROI 크기에 맞춰 **절대 좌표로 변환**
2. 변환된 기대 간격과 BFS 감지 간격 비교
3. 차이가 크면(±30% 이상) **스케일 팩터 계산** → 양식 힌트의 절대값들을 보정
   ```
   scaleFactor = 실제감지간격 / 양식기대간격
   보정된 bubbleW = 양식bubbleW × scaleFactor
   보정된 colSpacing = 양식colSpacing × scaleFactor
   ```
4. 보정된 값으로 Stage1.5~Stage4 진행

**위치**: `analyzeROI()` 초반부 patternHint 전처리 (새 코드)
**영향**: patternHint 값만 보정. 기존 Stage 로직 변경 없음.

---

## D 유형 교정: 노이즈 블롭으로 열 위치 밀림 (032, 172107_005)

**원인**: 행 내에 노이즈 블롭이 끼어서 열 평균 좌표가 밀림

**수정 — 양식 기대 간격으로 이상 블롭 제거**:
1. 양식의 기대 열간격 대비 **인접 블롭 간격이 50% 미만**이면 노이즈로 판정
2. 노이즈 블롭 중 크기가 작은 것 제거
3. B 유형의 크기 필터와 결합 가능 (같은 단계에서 처리)

**위치**: B 유형과 동일 위치 (Stage1 직후 노이즈 필터)
**영향**: B 유형과 함께 처리. 기존 로직 변경 없음.
