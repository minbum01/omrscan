# 현재 OMR 엔진 로직 정리 및 문제점

## 전체 흐름

```
사용자 박스 드래그
  → canvas.js: autoDetect() 호출 (방향/문항수/지선다 추정)
  → canvas.js: analyzeROI() 호출 (실제 판독)
    → findGrid() (BFS 블롭 탐지 + 그룹핑)
    → adjustPositions() (초과/부족 위치 조정)
    → sampleCell() (각 셀 픽셀 읽기)
    → 마킹 판별 (돌출도 비교)
```

---

## 1단계: 전처리 (preprocess) ✅ 안정적

- Min(R,G,B) → 채도 보정 → 밝기 정규화 → S-curve(k=10)
- **문제 없음**

---

## 2단계: findGrid (BFS + 그룹핑) ❌ 근본적 문제

### 하는 일
1. BFS로 어두운 영역(블롭) 탐지
2. 크기 필터: 면적 0.3~1.8배, w<40, h<40, 비율 0.4~2.5
3. Y기준 그룹핑 → 행 그룹 (허용 오차: medH * 0.5)
4. X기준 그룹핑 → 열 그룹 (허용 오차: medW * 0.5)
5. 노이즈 제거: 블롭 2개 미만 그룹 제거

### 반복되는 문제
| 문제 | 원인 | 발생 빈도 |
|------|------|----------|
| **4열이 3열로 감지** | 인접 열 합쳐짐 or 노이즈 필터로 제거 | 매우 잦음 (우하단 과목) |
| **문제번호가 열로 잡힘** | BFS가 숫자 블롭을 버블로 인식 | 매번 |
| **테두리가 열로 잡힘** | 시트 가장자리 선이 블롭으로 잡힘 | 우하단에서 |
| **가로/세로 오판** | 열 수 오류 → modeColCount 오류 → 방향 오판 | 자주 |
| **그룹핑 허용 오차 민감** | 0.75 → 합침, 0.5 → 분리 과다. 이미지마다 다름 | 항상 |

### 근본 원인
- **BFS는 "어두운 건 다 잡는다"** → 버블, 숫자, 테두리, 잉크 번짐 구분 불가
- 그룹핑 허용 오차가 고정값 → 이미지 해상도/DPI/버블 간격에 따라 맞을 수도 틀릴 수도
- 열 수가 틀리면 → 방향 판정, 위치 매핑, 마킹 판별 전부 연쇄적으로 틀어짐

---

## 3단계: adjustPositions ⚠️ 부분적 문제

### 하는 일
- 열 초과: 블롭 수 가장 많은 N개 선택
- 행 초과: 균등 간격 N개 연속 부분집합
- 부족: 기존 간격 기반 보간

### 문제
- 열이 이미 3개로 잘못 감지되면 → 4개를 맞출 수 없음 (보간으로 추정하지만 위치 부정확)
- 열 초과 시 "블롭 수 기반 선택"은 좋으나, findGrid가 열을 제대로 못 찾으면 무의미

---

## 4단계: sampleCell ✅ 안정적

- 로컬 적응형 임계값 (셀 평균의 75%)
- 중심부(50%) 별도 샘플링
- **문제 없음** (위치만 정확하면 잘 동작)

---

## 5단계: 마킹 판별 ✅ 개선됨

- 돌출도 비율(promRatio) > 1.5 → 마킹
- prom < 50 → 2차 검증 (50% 축소 샘플링)
- 중복 감지: pScore * 0.9 + fillDiff/inkDiff
- **위치가 정확하면 잘 동작**

---

## 6단계: autoDetect ❌ findGrid 의존

### 하는 일
- findGrid 호출 → numRows, numCols, modeColCount → 방향/문항/지선다 결정

### 방향 판정 로직
```
modeColCount > 5 → horizontal
numCols > numRows → horizontal
그 외 → vertical
```

### 문제
- findGrid가 열을 3개로 잡으면 → numCols=3, numRows=20 → vertical은 맞지만 numChoices=3 (실제 4)
- findGrid가 열을 6개로 잡으면 → modeColCount=6 > 5 → horizontal 오판 (실제 vertical)

---

## 핵심 결론

**findGrid의 BFS 그룹핑이 모든 문제의 근원.** 이후 단계(adjustPositions, sampleCell, 마킹 판별)는 위치만 정확하면 잘 동작함.

---

## 대안 검토

### A. findGrid 폐기 → ROI 균등 분할
- 사용자가 numQ, numC를 설정하면 ROI를 균등 분할
- **장점**: BFS 의존 제거, 항상 정확한 열 수
- **단점**: ROI에 문제번호/테두리 포함 시 균등 분할이 버블 위치와 안 맞음
- **보완**: 사용자가 ROI를 정확히 버블 영역만 감싸도록 유도 (이동/크기조절 가능)

### B. findGrid를 보조로만 사용
- 균등 분할 기본 + findGrid로 미세 보정
- findGrid가 실패해도 균등 분할로 폴백
- **장점**: 안정적 + 보정 가능
- **단점**: 구현 복잡도 증가

### C. findGrid 개선
- 그룹핑 허용 오차를 동적으로 (버블 간격 기반)
- BFS 대신 투영 기반 열 감지 (수직 투영 피크)
- **장점**: 근본적 해결
- **단점**: 이미 여러 번 시도했으나 완벽하지 않음

---

## 권장 방향

**A안 (균등 분할) + 수동 보정**

이유:
1. sampleCell + 마킹 판별은 이미 잘 동작함
2. 문제는 오직 "셀 위치를 어디로 잡느냐"
3. 사용자가 ROI를 정확히 잡으면 균등 분할이 가장 안정적
4. ROI 이동/크기조절 기능이 이미 있음
5. 자동 감지는 numQ/numC 추정에만 사용 (위치 결정에는 사용 안 함)
