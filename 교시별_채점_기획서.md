# 교시별 채점 기획서

## 📌 문제 정의

### 현재 한계
- 한 세션 = 한 OMR 양식 = 한 번의 시험
- OMR 양식(ROI 구성, 답안키)이 세션 전체에 공유됨 → **교시별로 양식이 다를 수 없음**
- 하루에 두 번 시험 보는 경우, 두 번째 OMR을 첫 번째 양식으로 채점해야 해서 **과목 구분 불가**
- 두 번째 OMR에 맞게 양식을 수정하면 첫 번째 OMR 결과가 **뒤엎힘**

### 원하는 것
- **한 세션 안에서 교시를 여러 개** 운영
- **교시마다 독립된 OMR 양식 / 답안키 / 과목 구성**
- 학생별로 모든 교시 결과를 **자동으로 연결** (수험번호/전화번호 기반)
- 최종 성적표는 **교시 통합**(같은 과목명은 합산, 교시 간 과목 중복 없음 가정)

---

## 🎯 핵심 원칙

1. **한 세션 = 여러 교시** (1교시, 2교시, ..., N교시)
2. **한 교시 = 고유한 OMR 양식** (ROI + 답안키 + 과목 설정)
3. **한 교시 = 여러 이미지** (학생별 OMR 한 장씩)
4. **한 학생 = 교시 수만큼 OMR** 존재 (결시도 허용)
5. **학생 매칭**: 수험번호(우선) → 전화번호 → 이름 (교시 간 학생 연결)
6. **과목 합산**: 같은 과목명은 교시가 달라도 합산 (기존 다과목 로직 그대로 확장)

## 📋 케이스 분석

### Case 1: 단일 교시, 단일 과목 (기존 기본 케이스)
- 1교시만 존재, 1 OMR = 1 과목
- 예: 고등학교 중간고사 국어 단일 시험
- 현재 시스템으로도 처리됨 → 1교시 1과목으로 자연스럽게 맵핑

### Case 2: 단일 교시, 다과목 (한 OMR에 여러 과목)
- 1교시만 존재, 1 OMR에 여러 subject_answer ROI
- 예: 공무원 시험(국어+영어+한국사 한 장)
- 현재 시스템으로도 처리됨 → 1교시 다과목

### Case 3: 다교시, 교시당 단일 과목 ⭐ 신규
- N교시 각각 독립 OMR, 교시당 과목 1개
- 예: 로스쿨 시험 (1교시 언어이해, 2교시 추리논증)
- **현재 불가** → 이번 기획의 핵심

### Case 4: 다교시, 교시당 다과목 ⭐ 신규
- N교시 각각 독립 OMR, 교시당 과목 여러 개
- 예: 1교시 국어+영어 (한 장 OMR), 2교시 수학+과학 (다른 OMR)
- **현재 불가** → 이번 기획의 핵심

### Case 5: 교시별 응시자 풀 차이 (선택/결시)
- 1교시 전체 응시, 2교시는 일부만 응시(선택과목 또는 결시)
- 학생 매칭 시 일부 교시에 OMR 없는 학생은 **해당 교시만 공란** 처리
- 성적표에서 해당 과목만 "미응시"로 표기

### Case 6: 같은 과목명이 여러 교시에 걸침 (합산?)
- 1교시 국어 10문항 + 2교시 국어 10문항 → 합산?
- **확정 필요** (아래 "⚠ 확정 필요 사항 참조")

### Case 7: 교시별 같은 OMR 양식 (재사용)
- 두 번의 시험이 같은 양식인 경우, 2교시가 1교시 양식을 복사해서 시작
- **편의 기능** (필수는 아님)

## 🏗 데이터 구조 설계

### 세션 (App.state)

```
session = {
  name: '2026 봄 모의고사',
  periods: [
    {
      id: 'p1',
      name: '1교시 (언어이해)',
      images: [img1, img2, ...],        // 이 교시 소속 OMR 이미지들
      answerKey: {...},                  // 이 교시 고유 답안키
      roiTemplate: {...},                // 이 교시 고유 ROI 템플릿(없을 수도 있음)
      subjects: {...}                    // 이 교시의 과목 설정
    },
    {
      id: 'p2',
      name: '2교시 (추리논증)',
      ...
    }
  ],
  currentPeriodId: 'p1',                 // 지금 편집/분석 중인 교시
  students: [...]                        // 세션 전체 응시자 (교시 공유)
}
```

### 이미지
```
image = {
  name, path, imgElement, ...,
  periodId: 'p1',                        // 어느 교시 소속
  rois: [...],                           // 교시별 독립 (교시 간 복사/공유 가능)
  results: [...],
  gradeResult: {...}
}
```

### collectData 출력 (교시 통합)
```
row = {
  examNo, name, birthday, phone,
  subjects: {
    '언어이해': { score, correctCount, ..., period: 'p1' },
    '추리논증': { score, correctCount, ..., period: 'p2' }
  },
  totalScore, totalCorrect, ...,
  _periodMissing: ['p2']                 // 결시한 교시 표기 (옵션)
}
```

## 🖼 UI 변화

### 상단 헤더
현재: `[세션명]`
변경: `[세션명] │ [1교시▼] [2교시] [+ 교시 추가]`
- 교시 드롭다운 또는 탭
- 현재 선택된 교시의 이미지/ROI만 보임
- "+ 교시 추가" 버튼으로 새 교시 생성

### 이미지 업로드
- 드래그 앤 드롭 시 **현재 선택된 교시**에 추가됨
- 기존 이미지를 다른 교시로 이동하는 UI도 있어야 함 (우클릭 메뉴 or 드래그)

### ROI 편집기
- 현재 교시의 이미지에 그린 ROI만 그 교시에 적용
- 다른 교시 이미지에는 영향 없음
- **템플릿 복사**: 한 교시의 ROI를 다른 교시로 복사하는 편의 기능

### 분석/채점
- 현재 교시 이미지만 분석/채점
- 또는 "전체 교시 일괄 채점" 버튼

### 채점 결과 (성적 탭)
- OMR 결과표: 교시별 드롭다운 or 통합 보기
- 성적일람표: 교시 구분 없이 과목별 열 반복 (기존 다과목 로직 그대로)
- 문항분석표: 교시·과목 드롭다운
- 개인별 성적표: 모든 교시 과목 통합 표시

## 🔄 영향 받는 로직 전체 점검

### 1. 세션 관리 (`session.js`)
- 세션 저장/로드 시 `periods` 배열 포함
- 기존 1교시 세션 마이그레이션 (자동으로 `periods=[{id:'p1', name:'1교시', images:기존이미지, ...}]`로 변환)

### 2. 이미지 관리 (`imageManager.js`)
- 현재 교시의 이미지만 리스트에 표시
- 업로드 시 `periodId` 자동 할당
- 이미지 이동/복사 기능

### 3. ROI 편집 (`canvas.js`, `ui.js`)
- 이미지에 ROI 그리면 그 이미지의 `periodId` 교시 설정에 저장
- 같은 교시 내 이미지 간 ROI 공유 (기존 동작 유지)
- 교시 전환 시 ROI 초기화되지 않음 (교시별 독립 보관)

### 4. 답안키 (`grading.js`)
- `App.state.answerKey` → `session.periods[p].answerKey`
- 답안키 편집 UI가 현재 교시의 답안키를 편집

### 5. 분석 파이프라인 (`canvas.js`, `batch.js`)
- 현재 교시 이미지만 분석
- 또는 전체 일괄 분석 (모든 교시 순회)

### 6. 채점 엔진 (`grading.js`)
- 교시별 답안키로 교시별 이미지 채점
- 결과는 이미지 단위로 `gradeResult`에 저장 (기존과 동일)

### 7. 집계 (`scoring.js`)
- **가장 큰 변화**: `collectData()`가 여러 교시의 이미지를 **수험번호로 매칭**해서 한 학생 row로 합침
- 학생 매칭 알고리즘 (교시 간):
  1. 수험번호 일치
  2. (수험번호 없을 때) 전화번호 일치
  3. (둘 다 없을 때) 이름 일치
  4. 매칭 실패 시 해당 교시 과목은 `null` (미응시 처리)
- 과목별 분석은 기존 로직 그대로 (subjects dict에 과목이 어느 교시에서 왔는지 관계없이 합산)

### 8. 성적표 (`scoring.js`)
- 성적일람표: 그대로 작동 (과목별 열 반복 + 총점)
- 개인별 성적표: 과목별 카드에 교시 표기 추가 ("국어 (1교시)")
- 문항분석표: 과목 드롭다운에 "모든 교시" 또는 교시별 선택 옵션

### 9. CSV (`scoring.js`, `subjectManager.js`)
- 시험인원 CSV: 세션 공유 (변화 없음)
- 과목/정답 CSV: 교시별로 로드할 수 있어야 함 (현재 교시의 과목만 편집)
- OMR 결과 CSV: 교시 정보 포함 or 과목만

### 10. 시험관리 모달 (`subjectManager.js`)
- 과목 관리 UI에 "현재 교시" 표시
- 또는 교시별로 과목 섹션 분리

## ⚠ 확정 필요 사항 (피드백 요청)

### 1. 교시별 같은 과목명 처리
- 1교시 "국어"와 2교시 "국어"가 있을 때:
  - (a) **합산** (한 학생의 국어 총점 = 1교시 국어 + 2교시 국어)
  - (b) **분리** ("국어_1교시" / "국어_2교시" 로 따로 표기)
  - (c) **금지** (같은 과목명은 다른 교시에서 쓰면 에러)

### 2. 교시 이름 기본값
- "1교시", "2교시", ... 로 자동 생성?
- 또는 사용자가 처음부터 입력 (예: "언어이해", "추리논증")?

### 3. 이미지와 교시의 관계
- (a) 이미지는 무조건 한 교시에 소속 (이동 가능)
- (b) 이미지를 여러 교시에 중복 사용 가능 (예: 같은 OMR을 두 번 채점)

### 4. 학생 매칭 순서
- 수험번호 > 전화번호 > 이름 순서 OK?
- 이름만 같고 수험번호 다르면 → 동명이인으로 **별도 학생** 취급?

### 5. 교시 삭제 시 동작
- 교시 삭제하면 그 교시의 이미지/결과 모두 삭제?
- 또는 이미지는 남기고 교시만 "미할당"으로?

### 6. 성적일람표 표기
- 과목 열 헤더에 교시 표시할지 ("1교시 국어" vs "국어")
- (기본: 과목명만, 툴팁으로 교시 보여주기?)

### 7. 시험관리 모달 과목 설정
- (a) 교시별로 완전 분리된 과목 목록
- (b) 통합 목록에서 각 과목이 어느 교시인지 태그
- (c) 교시 드롭다운 필터 + 통합 목록

### 8. 기존 세션 하위호환
- 기존 "다과목 1교시" 세션을 로드했을 때:
  - 자동으로 1교시로 감싸기 (투명 마이그레이션)
  - 사용자 경험 변화 최소화

### 9. 우선순위
- 전체 한 번에 구현 vs Phase별 점진 배포
- 제일 시급한 케이스는?
