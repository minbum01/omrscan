# OMR 버블 감지 문제 분석

## 사용자가 원하는 흐름

```
1. 박스를 친다 (ROI)
2. OMR 버블을 자동 인식한다
3. 자동인식이 안 되면 → 사용자가 버블 크기 슬라이더로 수동 지정
4. 슬라이더 조절 시 실시간으로 오버레이가 동기화됨 (찾은 버블이 보임)
5. 지정된 버블 크기를 기준으로 박스 안에서 버블들을 찾는다
6. 찾은 버블 위치에서 마킹 판별
```

## 현재 문제점

### 문제 1: 행마다 블롭 수가 다름 (4지선다인데 7개 나옴)

**원인**: `analyzeROI`에서 행 내 블롭 수를 제한하지 않음

```javascript
// 현재 코드 (omrEngine.js 247-251행)
for (let q = 0; q < Math.min(numQ, rowGroups.length); q++) {
    const rowBlobs = [...rowGroups[q]].sort(...);
    for (let c = 0; c < rowBlobs.length; c++) {  // ← rowBlobs 전부 사용!
```

사용자가 4지선다로 설정해도, 행에 블롭이 7개 있으면 7개 전부 처리함.
노이즈 블롭이나 잘못 그룹핑된 블롭이 포함되어 answer=7 같은 결과가 나옴.

**해결**: 행당 블롭 수를 numC로 제한. 블롭이 numC보다 많으면 가장 균등한 간격의 numC개를 선택.

### 문제 2: 버블 크기 슬라이더가 실시간 동기화 안 됨

**원인**: 슬라이더 조절 시 `this.bubbleSize` 값만 변경하고, 분석이 다시 실행되지 않음.
오버레이는 이전 분석 결과를 보여주고 있어서 변화가 안 보임.

```javascript
// 현재 코드 (canvas.js)
bubbleInput.addEventListener('input', () => {
    this.bubbleSize = parseInt(bubbleInput.value);
    bubbleVal.textContent = ...;
    // ← 여기서 끝! 분석 재실행 없음, render()도 없음
});
```

**해결**: 슬라이더 조절 시 → 즉시 분석 재실행 → 오버레이 갱신

### 문제 3: BFS가 빈 버블을 못 찾음

**원인**: BFS는 "어두운 픽셀 덩어리"를 찾음. 빈 버블(○)의 테두리가 연하면 임계값을 통과하지 못해 아예 감지 안 됨.
→ 한 행에 4개 버블 중 마킹된 1개만 감지 → 블롭 1개로 판단

**해결 한계**: BFS 방식의 근본적 한계. 버블 크기를 지정해도 "보이지 않는" 버블은 찾을 수 없음.
→ 진하기 슬라이더로 이미지를 진하게 하면 빈 버블 테두리도 감지될 수 있음.

### 문제 4: 행 그룹핑 오류 (기울어진 스캔)

**원인**: Y좌표 기준으로 행을 그룹핑하는데, 스캔이 기울어지면 같은 행의 블롭들이 서로 다른 Y좌표를 가짐.
→ 한 행의 블롭이 두 행에 나뉘거나, 두 행의 블롭이 한 행에 합쳐짐.

**해결**: 행 그룹핑 시 threshold를 블롭 높이가 아니라 행 간격의 일정 비율로 설정.

## 수정 계획

### 수정 1: 행당 블롭 수 = numC로 제한
- 블롭이 numC보다 많으면 → 가장 균등한 간격의 numC개 선택
- 블롭이 numC보다 적으면 → 있는 만큼만 사용 (빈 셀은 감지 불가)

### 수정 2: 버블 크기 슬라이더 실시간 동기화
- 슬라이더 input 이벤트 → 분석 재실행 → render
- debounce 적용 (200ms) → 슬라이더 빠르게 움직일 때 과도한 재실행 방지

### 수정 3: 균등 그리드와의 조합
- BFS로 찾은 블롭이 numQ×numC보다 적으면
  → 찾은 블롭 위치를 기반으로 나머지를 보간 (균등 분할)
  → "있는 건 실제 위치, 없는 건 추정 위치"
