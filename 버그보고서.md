# 버그 보고서 (2026-04-14)

전체 로직 점검 결과

---

## CRITICAL (즉시 수정 필요)

### BUG-1. ROI 타입 불일치 (phone_exam vs phone)
- **파일**: scoring.js, imageManager.js
- **내용**: ROI_TYPES에서 'phone_exam'을 'exam_no'+'phone'으로 분리했지만, scoring.js와 imageManager.js에 'phone_exam' 참조가 남아있음
- **영향**: 수험번호/전화번호 감지가 일부 경로에서 실패
- **수정**: 모든 'phone_exam' → 'exam_no' || 'phone' 통일

### BUG-2. gradeResult null 접근
- **파일**: scoring.js collectData
- **내용**: `img.gradeResult.score` 접근 시 gradeResult가 null이면 에러
- **영향**: 분석만 하고 채점 안 한 이미지에서 크래시
- **수정**: `img.gradeResult?.score || 0` 으로 변경

### BUG-3. 0으로 나누기 (percentage 계산)
- **파일**: scoring.js 개인별 성적표 렌더
- **내용**: `(s.score / max) * 100` 에서 max가 0일 때 Infinity
- **영향**: 만점이 0인 과목(미설정)에서 NaN/Infinity 표시
- **수정**: `max > 0 ? (score / max) * 100 : 0`

## HIGH (기능 오류)

### BUG-4. 과목별 점수 계산 오류
- **파일**: scoring.js collectData
- **내용**: 과목별 score를 gradeResult.details에서 추출하는 로직이 불안정
- **영향**: 다과목 시 과목별 점수가 정확하지 않을 수 있음
- **원인**: details의 score 필드가 채점 방식에 따라 없을 수 있음
- **수정**: gradeResult에서 직접 과목별 점수 계산 대신, ROI별 정답 수 × 배점으로 계산

### BUG-5. 세션 로드 시 교시(period) 지원
- **파일**: session.js, app.js
- **내용**: App._initPeriods() 호출이 집에서 추가된 코드에 있지만, 여기에서는 미구현
- **영향**: 교시 기능 사용 시 세션 로드 실패 가능
- **수정**: App._initPeriods가 없으면 무시하도록 가드 추가

### BUG-6. 일괄채점 시 과목 정답 로드 누락
- **파일**: batch.js
- **내용**: batch에서 채점 시 UI._loadAnswersFromSubject 호출하지만, 다과목 환경에서 모든 ROI에 대해 호출되는지 불확실
- **영향**: 일괄채점 시 일부 과목 정답 미적용
- **수정**: 채점 전 모든 subject_answer ROI에 대해 과목 정답 로드 보장

## MEDIUM (UX 문제)

- **수정**: 매칭 필드 기반으로 확장

### BUG-9. CSV 파싱 — 설명 텍스트 오인

- **내용**: 20행 이하 파일에서 fallback 시 설명 텍스트가 데이터로 잡힐 수 있음
- **영향**: 잘못된 과목/인원 등록
