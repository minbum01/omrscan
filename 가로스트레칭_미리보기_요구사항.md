# 가로 스트레칭 미리보기 요구사항 정리

## 현재 상태 (블롭확인오류.png 분석)

**보이는 것:**
1. **원본 ROI** (파란 박스): 왼쪽에 세로 길쭉 버블이 있는 OMR 영역
2. **미리보기** (주황 박스): ROI 우측에 가로로 2배 늘어난 이미지 — 버블이 동그랗게 보임 ✓
3. **BFS 디버그 블롭** (빨강/초록 작은 사각형): **원본 ROI 안에 표시되고 있음** ✗

**문제:**
- BFS 분석은 스트레칭된 이미지에서 실행됨 (맞음)
- 감지된 블롭 좌표를 **원본 좌표로 역변환**하여 원본 ROI 위에 그림
- 역변환된 블롭 크기가 가로로 압축되어 **원본의 좁은 버블 위에 표시**됨
- 사용자는 **감지가 제대로 되었는지 눈으로 확인할 수 없음**
  - 원본 버블은 여전히 길쭉해서 블롭 경계가 잘 안 보임
  - 미리보기(동그란 버블이 보이는 곳)에는 블롭 표시가 없음

---

## 사용자가 원하는 것

### 핵심 원칙
**"미리보기는 BFS가 제대로 동작하는지 검증하기 위한 시각 도구다"**

따라서 미리보기 안에 BFS 결과가 표시되어야 한다. 원본 ROI는 최종 채점 결과를 표시하는 곳이고.

### 기대 동작

1. **스트레칭 값 입력 + 미리보기 ON**
   - 원본 ROI 우측에 스트레칭된 이미지 표시 (✓ 지금 동작함)

2. **분석 실행**
   - BFS는 스트레칭된 미리보기 픽셀에서 실행됨
   - **감지된 블롭(빈 버블 사각형)은 미리보기 위에 그려짐**
     - 사용자는 동그란 버블들이 초록 사각형으로 제대로 감지되는지 눈으로 확인
     - "아, 이제 잘 잡히네" 하고 검증 가능
   - **마킹 판별 결과(체크된 버블)도 미리보기 위에 표시**
     - 초록=정답, 빨강=오답, 주황=빈칸 등

3. **동시에 원본 ROI에도 결과 표시 (최종 채점용)**
   - 원본 좌표로 역변환된 블롭 위치에 결과 오버레이
   - 이건 기존 로직 그대로 유지
   - 원본과 미리보기 둘 다 표시됨

### 요약

| 구분 | 원본 ROI | 미리보기 |
|------|---------|---------|
| 이미지 | 원본 (세로 길쭉 버블) | 스트레칭 (동그란 버블) |
| BFS 디버그 블롭 | 표시 안 함 (또는 최소) | **크고 명확하게 표시** |
| 마킹 판별 결과 | 표시 (채점 확인용) | **표시 (BFS 검증용)** |
| 좌표계 | 원본 절대 좌표 | 미리보기 영역 상대 좌표 |

---

## 구현 방향 (요약)

1. 분석 시 BFS 결과를 **두 좌표계로 저장**
   - `blobs.cx/cy` = 원본 절대 좌표 (역변환된 값)
   - `blobs.previewCx/previewCy` = 미리보기 영역 내 절대 좌표
2. 렌더링 시
   - 원본 ROI: 기존처럼 `cx/cy` 사용
   - 미리보기 영역: `previewCx/previewCy` 사용하여 오버레이 그리기
3. 디버그 블롭 (`debugBlobs.all`, `debugBlobs.filtered`)도 동일하게 두 좌표계 저장

---

## 주의사항
- 미리보기는 **ROI 우측 외부** 공간에 그리므로 다른 ROI나 원본 이미지 픽셀을 덮어쓰지 않음
- 미리보기 위치(`roi._previewRect`)는 render() 시 계산되므로 analyze 시점에는 없을 수 있음
  → 분석 시 `previewRect`를 미리 계산해서 저장해둬야 함
- 미리보기 체크박스 OFF 상태에서는 미리보기 오버레이 자체가 없으므로 블롭도 안 보임
  → 분석 시점에 미리보기가 OFF였어도 블롭 좌표는 저장해둬서, 나중에 켜면 표시되도록
