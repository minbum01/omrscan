# 개인별 성적표 기획서

## 🎯 목표
- **A4 가로 (297mm × 210mm) 1~2페이지**에 한 학생의 시험 결과를 **최대한 많은 데이터**로, **현대적인 디자인**으로 표현
- 인쇄 출력 + PDF 저장 + 화면 뷰 모두 깨지지 않아야 함
- 다과목 채점 결과를 종합적으로 보여주는 최종 산출물

## 🔖 참고 레퍼런스
- **SAT Score Report** (College Board) — 섹션별 점수 + 전국 백분위 + 강점/약점 분석
- **TOEIC / IELTS 성적표** — 레벨 밴드 + 스킬별 프로파일
- **한국 대학입시 성적표 (수능/모의고사)** — 과목별 표준점수/백분위/등급 그리드
- **Fitbit / Apple Health 리포트** — 카드형 대시보드 + 게이지 + 마이크로차트
- **Stripe / Linear 대시보드** — 고밀도 정보를 여백으로 구조화하는 현대 UI

## 🧱 구성 요소 상세

### ① HEADER BAND
| 요소 | 내용 | 출처 |
|------|------|------|
| 시험명 | SessionManager.currentSessionName | 세션 |
| 시행일 | 시험 실시일 (세션 메타) 또는 발행일 | 세션/오늘 |
| 학생명 (대형) | `row.name` | collectData |
| 발행 정보 | 발행 기관명 (설정 가능) | 설정 |

**스타일**: 진한 배경(Navy #0f172a or Deep Blue), 흰색 텍스트, 기울임 없는 굵은 산세리프

### ② IDENTITY STRIP
학생을 식별하는 모든 메타 필드를 한 줄로 압축. 기타 ROI(`etcFields`)는 정의된 순서대로 이어 붙임.

| 필드 | 값 |
|------|-----|
| 응시번호 | `row.examNo` |
| 성명 | `row.name` |
| 생년월일 | `row.birthday` |
| 전화번호 | `row.phone` |
| 과목코드 | `row.subjectCode` |
| 파일명 | `row.filename` (디버그 시 표시) |
| 기타 N | `row.etcFields[name]` |

**스타일**: 라벨 회색 9pt + 값 검정 11pt 볼드, 세로 구분선

### ③ 종합 요약 (좌측 패널, 폭 ~90mm)

**대형 점수 블록**
```
╔══════════════════╗
║    172 / 300     ║   ← 총점 / 만점 (48pt Bold)
║                  ║
║     57.3%        ║   ← 만점 대비 백분율 (20pt)
╚══════════════════╝
```

**등급 밴드** (설정 가능한 구간)
```
[ S ][ A ][ B ][ C ][ D ][ E ]
        ▲
       나
```
- 기본 구간: 90+/80+/70+/60+/50+/under
- 나중에 사용자 커스터마이징 (과목별 다른 기준도 가능)

**핵심 지표 카드 (2×3 그리드)**
| 석차 | 5 / 60명 |
| 상위 % | 상위 8.3% |
| 백분위 | 91.7 |
| 표준점수 | 63.4 |
| 평균 | 142.6 |
| 평균 대비 | +29.4 ▲ |

**스타일**: 각 지표는 좌측 컬러 액센트 바 (#3b82f6 등), 값은 20pt Bold

---

### ④ 과목별 성적 그리드 (우측, 폭 ~190mm)

과목 수에 따라 카드 배치:
- 1과목 → 카드 1장 (전폭)
- 2~3과목 → 가로 3열
- 4과목 → 2×2 그리드
- 5과목 이상 → 스크롤 없이 축소 or 2페이지 분할

**과목 카드 구조 (1장)**
```
┌─────────────────────────────┐
│  🟦 국어       100점 만점   │  ← 과목명 + 만점 (헤더 컬러 구분)
│  ─────────────────────────  │
│       92 점                 │  ← 대형 점수 (36pt)
│  ████████████░  92%         │  ← 달성률 progress bar
│  ─────────────────────────  │
│  맞은 개수  23 / 25         │
│  틀린 개수  2               │
│  석차      3위 / 60명       │
│  백분위    95.0             │
│  표준점수  68.2             │
│  평균      78.4 (Δ +13.6)  │
│  상위 27%   88점 이상        │  ← 기준 참고
└─────────────────────────────┘
```

**색상 로직**: 점수 달성률에 따라 헤더 색상 변함 (90+: 초록, 70+: 파랑, 50+: 주황, under: 빨강)

---

### ⑤ 시각화 바 (높이 ~40mm, 3분할)

#### 5-1. 과목별 점수 비교 막대
- X축: 과목명
- Y축: 점수 (만점 기준 정규화 0~100%)
- 막대 3종: `내 점수` (파랑, 굵게) / `평균` (회색) / `만점` (옅은 배경)
- 데이터 라벨: 각 막대 위 숫자

#### 5-2. 전체 분포 히스토그램
- X축: 총점 구간 (10점 bin)
- Y축: 인원
- 내 점수 위치에 빨간 수직선 + "나" 라벨
- 평균 위치에 회색 점선

#### 5-3. 표준점수 레이더 (3과목 이상) / 정답률 도넛 (1~2과목)
- **레이더**: 각 축 = 과목, 외곽 = 표준점수 70, 중심 = 30
  - 내 점수 폴리곤 파랑
  - 평균(=50) 기준선 회색 원
- **도넛**: 과목이 1~2개일 때는 정답률 도넛 (맞은수 / 만점 문항수)

---

### ⑥ 문항별 정오표 (과목별 그리드)

과목마다 한 줄 (또는 문항이 많으면 2줄):
```
국어  ①② ③④ ⑤⑥ ⑦⑧ ⑨⑩ ⑪⑫ ⑬⑭ ⑮⑯ ⑰⑱ ⑲⑳ …
      O  O  X  O  O  ∅  O  O  X  O  O  O  O  O  …
```

**표현 방식**:
- 각 문항 셀: 20×24pt 박스, 상단에 문항번호, 하단에 O/X/∅
- 색상: O=#dcfce7(초록 배경), X=#fee2e2(빨강 배경), ∅=#f1f5f9(회색)
- **매력적 오답 표시**: X인데 매력적 오답 규칙에 걸리면 주황 테두리 + 우측상단 작은 ⚠

---

### ⑦ 하단 푸터
```
발행: 2026-04-13 19:21 · 문서번호: OMR-20260001-001 · 엔진 ver 1.0  · Page 1 of 1
```

## 📄 페이지 2 (선택적 상세)

### 오답 분석
**틀린 문항 표**
| 과목 | 문항 | 내 답 | 정답 | 난이도(정답률) | 변별도 | 비고 |
|------|------|-------|------|----------------|--------|------|
| 국어 | 3 | 2 | 4 | 54.2% | 0.43 | 매력적 오답 |
| 국어 | 9 | ∅ | 1 | 78.3% | 0.21 | 미응답 |
| 영어 | 15 | 3 | 5 | 32.8% (난) | 0.55 | |
| ... | | | | | | |

### 강점 / 약점 분석
```
💪 강점
- 국어: 평균 대비 +13.6점, 표준점수 68 (상위 5%)
- 영어: 만점 문항 중 쉬운 문항 전 정답 (난이도 80%↑)

⚠ 약점
- 한국사: 평균 대비 -4.2점, 상위 27% 내 미포함
- 변별도 높은 문항 중 오답률 높음 (50%)
```

**생성 로직** (자동화 가능한 것만):
- 평균 대비 +5점 이상 과목 → 강점
- 평균 대비 -5점 이상 또는 하위 27% 그룹에 속한 과목 → 약점
- 쉬운 문항(정답률 70%+) 오답 → "기본기 점검 필요"
- 변별도 높은 문항(0.4+) 오답 → "도전 문항 미해결"

## 🗄 데이터 요구사항

### 이미 수집된 데이터 (collectData + calcStats + calcItemAnalysis)
- ✅ `row.name`, `row.examNo`, `row.birthday`, `row.phone`, `row.subjectCode`, `row.filename`, `row.etcFields`
- ✅ `row.totalScore`, `row.totalCorrect`, `row.totalWrong`, `row.totalMax`
- ✅ `row.rank`, `row.tScore`, `row.percentile` (총점 기준)
- ✅ `row.subjects[과목].score / correctCount / wrongCount / totalPossible`
- ✅ `row.subjects[과목].rank / tScore / percentile`
- ✅ `row.answers` 전체 + `row.subjects[과목].answers` (q, marked, markedLabel, isCorrect, correctAnswer)
- ✅ `calcStats(rows)` → N, mean, stdDev, max, min
- ✅ `calcItemAnalysis(rows, subj)` → 문항별 정답률/변별도/매력적 오답

### 추가로 필요한 계산
- 📌 **과목별 stats** (mean, stdDev, max, min): 현재 `calcStats`는 총점만 반환. 과목별도 필요 → 확장
- 📌 **등급 구간** (S/A/B/C/D/E): 설정 가능한 기준 또는 기본값 고정
- 📌 **분포 히스토그램 bin**: 전체 rows에서 10점 단위 카운트
- 📌 **강점/약점 라벨**: 위 "생성 로직" 참조
- 📌 **매력적 오답 정보** per 문항: `calcItemAnalysis`의 반응분포에서 도출

## 📋 구현 순서 (제안)

### Phase 1 — 골격 (이번 차수 권장 범위)

## ❓ 확정 필요 사항 (사용자 피드백 요청)

1. **등급 구간**: 고정 기본값(90/80/70/60/50)? 사용자가 바꿀 수 있게? 과목별 다른 기준?
2. **페이지 수**: 1페이지 필수 / 2페이지는 선택인가, 아니면 항상 2페이지?
3. **시각화 3종** 중 우선순위? (막대 > 히스토그램 > 레이더 순으로 진행 권장)
4. **로고/기관명 표시**: 세션에 기관명 필드를 추가해서 표시할지, 아니면 생략할지?
5. **PDF 저장**: Electron `printToPDF` 자동 파일명(`이름_수험번호.pdf`)? 사용자 선택?
6. **일괄 PDF**: 전체 학생을 한 파일(여러 페이지)로? 학생당 개별 파일?
7. **QR/문서번호**: 현재 단계에서 구현? 나중으로 미룰지?
