# 세로 길쭉 버블 감지 방안 (10+)

## 현재 문제
- 기존 BFS 블롭 감지는 원형 버블 전제 (종횡비 0.7~1.4)
- 세로 길쭉 버블(종횡비 ~0.5)은 필터에서 탈락하거나, 테두리 파편만 잡힘
- 버블 간 세로 간격이 좁아 BFS에서 인접 버블이 합쳐지기도 함

---

## 방안 1: 버블 샘플 캘리브레이션 (사용자 수동)

**방법**: 사용자가 버블 한 줄을 드래그 → 선택지 수 입력 → 등분 계산으로 그리드 좌표 생성. 감지 없이 순수 계산.

**장점**: 어떤 형태의 버블이든 작동. 테두리/노이즈 무관.  
**단점**: 사용자 개입 필요. 줄이 살짝 기울어져 있으면 오차 누적.  
**난이도**: 낮음  
**핵심**: 감지를 포기하고 사용자가 기준을 알려주는 것

---

## 방안 2: 프로젝션(투영) 기반 그리드 탐색

**방법**: ROI를 가로/세로로 투영 → 어두운 픽셀 밀도 프로파일 → 주기적 피크 = 행/열 위치. BFS 없이 그리드 직접 생성.

**장점**: 블롭 형태 무관 (원형이든 길쭉이든). 규칙적 그리드에 강함.  
**단점**: 테두리 선이 강한 피크를 만들어 오탐 가능. 불규칙 그리드에 약함.  
**난이도**: 중  
**핵심**: 버블을 개별 찾지 않고, 그리드의 주기성을 이용

---

## 방안 3: 종횡비 적응형 BFS (2-pass)

**방법**:
1. 1st pass: 종횡비 제한 없이 BFS 블롭 전부 수집
2. 블롭들의 중앙값 종횡비 계산
3. 2nd pass: 중앙값 ± 허용 범위로 필터링

**장점**: 기존 BFS 로직 거의 그대로. 원형/길쭉 자동 대응.  
**단점**: 테두리/텍스트 블롭이 1st pass에서 함께 잡혀 중앙값 왜곡 가능.  
**난이도**: 낮음  
**핵심**: 종횡비를 하드코딩하지 않고 데이터에서 추론

---

## 방안 4: 템플릿 매칭 (Cross-Correlation)

**방법**: 샘플 버블 하나의 픽셀 패턴을 템플릿으로 저장. 전체 ROI에서 정규화 교차상관(NCC)으로 유사한 위치 탐색. 상관도 > 임계값인 위치 = 버블.

**장점**: 버블 형태에 정확히 맞춤. 노이즈/테두리 자동 무시.  
**단점**: 템플릿을 어떻게 얻느냐 (사용자가 찍거나, 자동 추출). 크기 변화에 민감. 계산량 많음.  
**난이도**: 중~높  
**핵심**: "이렇게 생긴 걸 찾아라"고 직접 보여주는 것

---

## 방안 5: 주파수 분석 (FFT/자기상관)

**방법**: ROI 이미지를 FFT 변환 → 주파수 도메인에서 그리드의 반복 주기(행 간격, 열 간격)를 추출 → 역변환으로 그리드 위치 계산.

**장점**: 회전/기울기에도 주기성은 유지됨. 노이즈에 강함.  
**단점**: 구현 복잡. 비균일 그리드에 약함. JavaScript에서 FFT 라이브러리 필요.  
**난이도**: 높음  
**핵심**: 그리드는 주기적 패턴 → 주파수 도메인에서 명확히 보임

---

## 방안 6: 모폴로지(형태학) 전처리 + BFS

**방법**:
1. 이진화 후 세로 방향 Erosion → 세로로 연결된 인접 버블 분리
2. Opening/Closing으로 노이즈 제거
3. 분리된 블롭에 기존 BFS 적용

**장점**: 합쳐진 버블 문제 해결. 기존 BFS 재활용 가능.  
**단점**: Erosion 커널 크기 조절 필요. 얇은 버블이 Erosion으로 사라질 수 있음.  
**난이도**: 중  
**핵심**: BFS 전에 전처리로 길쭉 버블의 연결 문제를 해결

---

## 방안 7: 세로 스트립 분할 분석

**방법**:
1. ROI를 세로 스트립(열)으로 나눔 (프로젝션으로 열 위치 추정)
2. 각 스트립 내에서 가로 방향 1D 프로젝션 → 행 위치 탐색
3. 열과 행의 교차점 = 버블 셀

**장점**: 열마다 독립 처리 → 일부 열이 손상되어도 나머지 동작. 테두리를 열 단위로 제거 가능.  
**단점**: 열 위치를 먼저 알아야 함. 세로 프로젝션에서 열 분리가 안 되면 실패.  
**난이도**: 중  
**핵심**: 2D 문제를 1D 문제 2번으로 분해

---

## 방안 8: 에지 감지 + 윤곽선 분석

**방법**: Sobel/Canny 에지 감지 → 폐쇄 윤곽선(contour) 추출 → 윤곽선의 면적/종횡비/볼록도로 버블 필터링.

**장점**: 채워진 버블과 빈 버블 모두 감지 가능 (윤곽선이 있으면 됨). 종횡비에 유연.  
**단점**: JavaScript에서 Canny 구현 필요. 노이즈가 많으면 가짜 윤곽선 다수.  
**난이도**: 중~높  
**핵심**: 블롭(영역)이 아니라 윤곽선(경계)을 찾는 접근

---

## 방안 9: Run-Length 기반 주기 탐색

**방법**:
1. 이진화된 이미지의 각 행(수평 스캔라인)에서 dark run 길이와 위치 기록
2. 여러 행에서 동일한 x 위치에 반복되는 dark run = 세로 버블
3. 반복 패턴에서 열 위치 추출 → 행도 동일하게 세로 스캔라인으로

**장점**: 메모리 효율적. BFS 없이 선형 스캔. 매우 빠름.  
**단점**: 기울어진 이미지에 약함. 구현이 까다로움.  
**난이도**: 중  
**핵심**: 픽셀 단위 스캔으로 반복 패턴을 직접 추출

---

## 방안 10: 그리드 피팅 (Grid Fitting)

**방법**:
1. ROI에서 아무 조건 없이 블롭 N개 추출 (크기/종횡비 무시)
2. 이 블롭 좌표들에 가장 잘 맞는 정규 그리드(rows × cols)를 최적화 (RANSAC 또는 최소자승법)
3. 그리드에 맞지 않는 블롭 = 노이즈로 제거

**장점**: 블롭 형태 완전 무관. 노이즈/테두리 자동 제거. 기울기 보정도 가능.  
**단점**: 최적화 알고리즘 구현 복잡. 블롭 수가 적으면 피팅 불안정.  
**난이도**: 높음  
**핵심**: "이 점들 중에서 그리드를 이루는 것만 골라라"

---

## 방안 11: 열 위치 고정 + 행별 1D 스캔

**방법**:
1. 버블 샘플(방안 1)로 열 위치(x좌표)만 확보
2. 각 열 위치에서 세로로 1D 밝기 프로파일 추출
3. 프로파일의 어두운 구간 = 해당 열의 버블 행 위치
4. 행/열 교차 → 그리드 완성

**장점**: 샘플에서 열만 알면 됨(행은 자동). 테두리 영향 없음 (열 위치가 테두리와 다르므로).  
**단점**: 사용자가 샘플 한 줄은 찍어야 함.  
**난이도**: 낮~중  
**핵심**: 방안 1(샘플) + 방안 7(스트립)의 하이브리드

---

## 방안 12: 적응형 임계값 + 연결 요소 라벨링

**방법**: 전역 임계값 대신 로컬 적응형 임계값(예: 블록 평균) 사용 → 이진화 품질 향상 → 연결 요소 라벨링으로 블롭 추출. 종횡비 필터 완화.

**장점**: 조명 불균일, 스캔 품질 저하에 강함. 기존 파이프라인과 유사.  
**단점**: 적응형 임계값 블록 크기 설정이 중요. 세로 연결 문제는 여전히 존재할 수 있음.  
**난이도**: 중  
**핵심**: 전처리 품질을 올려서 BFS 결과를 개선
