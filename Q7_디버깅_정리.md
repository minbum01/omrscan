# Q7 인식 오류 디버깅 정리

## 현재 상황
- 송다영 답안지의 **Q7(정답: 2번)**이 지속적으로 오인식됨
- Q7만 단독 박스 → **정상 인식**
- Q6~Q10 박스 → Q7이 **4번**으로 인식
- Q6~Q8 박스 → Q7이 **3번**으로 인식
- Q1~Q20 전체 → Q7이 **3번**으로 인식

---

## 시도한 것들과 결과

### 시도 1: 전처리 추가 (밝기 정규화 + S-curve 대비 강화)
- **의도**: 마킹과 빈 버블의 차이를 극대화
- **결과**: 대비가 너무 강해져서 행 사이 미세한 갭까지 검정으로 변환 → 문제 악화
- **S-curve k값**: 8 → 10 → 6으로 조정했으나 근본 해결 안 됨

### 시도 2: 채도 보정 (파란잉크 감지)
- **의도**: 색잉크 마킹을 더 어둡게 변환
- **결과**: 이 답안지는 검정 마킹이라 채도 보정 무관. 효과 없음

### 시도 3: 중심부 채움률(centerFillRatio) 추가
- **의도**: 마킹된 버블은 내부가 꽉 차있으므로 빈 테두리와 구분
- **결과**: 점수 공식에 반영됨. 마킹 버블이 **검출되면** 잘 작동하나, 블롭 자체가 안 잡히면 무용

### 시도 4: filterBlobs 완화 (크기 120px, 면적 200%, 어두운 블롭 보호)
- **의도**: 마킹 블롭이 커도 필터에서 보호
- **결과**: 글자, 테두리, 손글씨까지 전부 버블로 인식 → 동그라미가 이미지 전체에 산재

### 시도 5: filterBlobs 엄격 복원 + 탈락 블롭 복구
- **의도**: 필터는 엄격하게 유지하되, 탈락 중 마킹된 것만 복구
- **결과**: 복구 대상이 candidates(1차 통과)에서만 찾아서 실패 → 1차에서 이미 탈락한 블롭은 복구 안 됨

### 시도 6: 복구 대상을 전체 원본 블롭으로 확장
- **의도**: 1차 크기 필터 탈락 블롭도 복구 대상에 포함
- **결과**: 복구 **성공** — 18x36 블롭(Q6+Q7 합쳐진 것) 복구됨
- **새 문제**: 이 18x36 블롭이 **행6에 배정**되어 행7은 여전히 3블롭

### 시도 7: BFS 크기 제한 (MAX_BLOB_DIM = 50px)
- **의도**: BFS가 50px 넘게 확장 못 하게 제한
- **결과**: 제한은 적용되었으나, Q6과 Q7의 마킹이 물리적으로 연결된 상태라 50px 이내에서도 18x36 블롭이 생성됨

---

## 콘솔 로그 분석 (최신)

```
[복구] 18x36 area=648 ink=0.565 fill=0.690  ← Q6+Q7 합쳐진 블롭

행6 (4블롭): [2] score=868 18x36  ← 복구된 블롭이 행6에 배치됨
행7 (3블롭): 350, 596, 602       ← 마킹 블롭 없음 → answer=3 (오답)
```

### 핵심 발견
- Q6의 마킹(3번)과 Q7의 마킹(2번)이 **세로로 인접**해 있음
- 두 마킹 사이의 간격이 매우 좁아서 BFS가 하나의 블롭(18x36)으로 합침
- 이 합쳐진 블롭의 중심 Y가 Q6에 가까워 **행6에 배정**
- 결과: Q6은 4블롭(정상), Q7은 3블롭(마킹 누락)

---

## 근본 원인

**BFS가 세로로 인접한 두 마킹을 하나의 블롭으로 합친다.**

OMR 시트에서 같은 열에 위아래로 연속 마킹된 경우, 잉크가 번지거나 인쇄 품질에 따라 두 마킹 사이 갭이 1~2px 밖에 안 될 수 있다. 전처리(대비 강화)로 이 갭마저 검정이 되면 BFS가 연결해버린다.

## 아직 시도하지 않은 해결책

### A. 대형 블롭 분할
- 높이가 중간값의 1.5배 이상인 블롭을 수평으로 분할
- 블롭 내부에서 가장 밝은 수평 라인을 찾아 그 지점에서 자름
- → 18x36 블롭 → 18x17 + 18x17 두 개로 분리

### B. 전처리 제거/약화
- processImage에서 S-curve 대비 강화를 제거하거나 매우 약하게 (k=3)
- 대비 강화 없이 밝기 정규화만 적용
- → 행 사이 갭이 보존되어 BFS 합침 방지

### C. BFS 대신 그리드 기반 접근
- ROI를 행×열 그리드로 미리 나누고, 각 셀을 독립적으로 분석
- 사용자가 문항수와 지선다를 설정하므로 그리드 크기를 알 수 있음
- → BFS 합침 문제 자체가 발생하지 않음

### D. BFS 후 대형 블롭의 중심 Y를 기준으로 행별 분배
- 합쳐진 블롭의 Y 범위를 확인하여 여러 행에 걸치면 각 행에 가상 블롭 생성

---

## 제안하는 우선순위
1. **B** (전처리 약화) — 가장 빠르고 부작용 적음
2. **A** (대형 블롭 분할) — B로 안 되면 시도
3. **C** (그리드 기반) — 가장 근본적이지만 대규모 변경 필요
