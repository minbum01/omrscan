# 0324 수정 기획

## 버그 수정

### BUG-1: 분석 실행 후 우측 사이드바 갱신 안 됨
- **현상**: OMR 분석 실행 후 우측 패널의 결과가 갱신 안 됨
- **원인**: `canvas.js` runAnalysis()에서 분석 완료 후 `UI.updateRightPanel()` 호출이 없음
- **수정**: 분석 완료 후 `UI.updateRightPanel()` 호출 추가

### BUG-2: 4지선다인데 answer=7이 나옴
- **현상**: 사용자가 4지선다로 설정해도 행에 블롭 7개가 잡히면 7개 전부 사용
- **원인 1**: `omrEngine.js` analyzeROI()에서 행 내 블롭 수를 numC로 제한하지 않음
- **원인 2**: 버블 크기 필터가 ±50%로 너무 넓어서 노이즈도 통과함
- **수정**: 아래 IMP-1, IMP-4 참조

### BUG-3: 버블 크기 슬라이더 조절 시 실시간 동기화 안 됨
- **현상**: 슬라이더를 움직여도 화면에 변화 없음
- **원인**: 슬라이더 input 이벤트에서 값만 변경하고 분석 재실행/render가 없음
- **수정**: 슬라이더 변경 시 → debounce 300ms → 분석 재실행 → render + 우측패널 갱신

## 기능 개선

### IMP-1: 행 내 블롭 수 = numC 제한
사용자가 numC(지선다)를 설정한 경우:
- 행에 블롭 > numC: 가장 균등한 간격의 numC개 선택 (나머지는 노이즈로 판단)
- 행에 블롭 = numC: 그대로 사용
- 행에 블롭 = numC - 1: 간격 패턴으로 빠진 1개 위치 추정 → 보간 추가
  - 예: 4지선다인데 3개만 감지됨 → 간격에서 비어있는 위치를 찾아 1개 추가
- 행에 블롭 < numC - 1: 보간 불가 → 있는 것만 사용

### IMP-2: 버블 크기 슬라이더 실시간 동기화
- 슬라이더 조절 → 300ms debounce → 현재 이미지 분석 재실행
- 분석 결과 오버레이(초록 사각형)가 실시간으로 변함
- 사용자가 눈으로 보면서 최적 크기를 찾을 수 있음
- 슬라이더 값이 0이면 "자동" (기존 중간값 기반)

### IMP-3: 버블 크기 양식 저장
- 양식 저장 시 `settings.bubbleSize`에 슬라이더 값 포함
- 양식 불러오기 시 해당 값으로 슬라이더 설정
- 일괄채점 시 각 이미지에서 저장된 bubbleSize로 BFS 실행 → 이미지별 블롭 위치 새로 찾음
- 좌표는 저장하지 않음 (이미지마다 다르므로)

### IMP-4: 버블 크기 필터를 타이트하게
- 현재: bubbleSize ±50% → 20px 지정 시 10~30px 통과 (너무 넓음)
- 수정: bubbleSize ±15% → 20px 지정 시 17~23px 통과
- 버블만 정확히 걸러내고 글자/선/노이즈는 크기가 달라서 탈락
- 자동 모드(bubbleSize=0)에서는 기존 중간값 ±범위 유지

### IMP-5: 버블 모양 필터 (크기만이 아니라 형태도 검증)
BFS가 이미 알고 있는 정보로 "동그라미인지" 추가 검증:
- **종횡비**: 현재 0.3~3.0 → **0.7~1.4**로 (버블은 원형이므로 가로세로 비슷)
- **채움률**: `블롭픽셀수 / (w × h)` > 0.3 이상 (원은 바운딩박스의 ~78% 채움, 노이즈는 희박)
- 이 두 조건으로 같은 크기여도 글자/선/얼룩은 탈락, 동그라미만 통과

- 양식 저장에서 grid 좌표 저장은 제거 완료됨 (이전 작업)
- BFS 기본 로직은 유지 (버블 크기 필터만 개선)
