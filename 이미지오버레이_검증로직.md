# 이미지 오버레이 교차 검증 로직

## 개요

현재 OMR 감지는 **BFS 블롭 감지 → sampleCell → 복합스코어** 파이프라인으로 동작한다.
이 로직은 버블을 "찾아서" 분석하는 방식이므로, 버블을 못 찾으면 실패한다.

**3번째 검증 레이어**로 "이미지 오버레이 기준 계산"을 추가한다.
이 방식은 버블을 찾지 않고, ROI 영역을 문항수 × 선택지수로 **균등 분할**하여
각 구간의 검은색 비율만으로 마킹을 판별한다.

두 결과가 일치하면 → **확정(통과)**
두 결과가 불일치하면 → **별색 처리(교사 확인 필요)**

## 3단계 감지 구조

```
Layer 1: 기본 OMR (BFS 블롭 → sampleCell → 공식E)
         ↓ 동그란 버블에 최적화
         
Layer 2: 세로길쭉 OMR (BFS 길쭉필터 → sampleCell → 공식E-v4)
         ↓ h/w ≥ 1.4 길쭉 버블에 최적화

Layer 3: 이미지 오버레이 검증 (균등 그리드 분할 → 구간 평균 밝기 → 교차 검증)
         ↓ 버블 위치와 무관하게 영역 기반 분석
         ↓ Layer 1/2 결과와 비교 → 일치=확정, 불일치=별색
```

## Layer 3 상세 로직

### 1단계: 균등 그리드 분할

```
ROI 영역 (x, y, w, h)
├── 문항 수: numQ (예: 20)
├── 선택지 수: numC (예: 5)
├── 방향: vertical / horizontal

세로 방향일 때:
  행 높이 = h / numQ
  열 너비 = w / numC
  
  grid[q][c] = {
    x: ROI.x + c * (w / numC),
    y: ROI.y + q * (h / numQ),
    w: w / numC,
    h: h / numQ
  }
```

### 2단계: 구간별 검은색 비율 계산

```
각 grid[q][c]에 대해:
  1. 해당 영역의 ImageData 추출
  2. 그레이스케일 변환
  3. 영역 전체 평균 밝기 계산
  4. 임계값(Otsu 또는 고정) 이하 픽셀 비율 = darkRatio
  
결과: overlayScores[q][c] = darkRatio (0.0 ~ 1.0)
```

### 3단계: 행별 마킹 판별

```
각 행(문항)에서:
  1. overlayScores[q]를 정렬
  2. 1위와 2위의 차이(gap) 계산
  3. gap ≥ 임계값 → overlayAnswer = 1위 선택지
  4. gap < 임계값 → overlayAnswer = null (불확실)
```

### 4단계: 교차 검증

```
각 문항에 대해:
  bubbleAnswer = Layer 1/2의 결과 (BFS 기반)
  overlayAnswer = Layer 3의 결과 (그리드 기반)
  
  Case A: bubbleAnswer === overlayAnswer
    → 확정 ✓ (두 방법이 일치 → 높은 신뢰도)
    
  Case B: bubbleAnswer !== null && overlayAnswer !== null && 서로 다름
    → 별색 처리 ⚠ (충돌 → 교사 확인 필요)
    
  Case C: bubbleAnswer !== null && overlayAnswer === null
    → 부분 확정 △ (BFS만 감지, 오버레이 불확실)
    
  Case D: bubbleAnswer === null && overlayAnswer !== null
    → 오버레이 참조 가능 (BFS 실패, 오버레이 감지)
    
  Case E: 둘 다 null
    → 미기입 ○ (둘 다 감지 못함 → 빈칸)
```

## 장점

1. **BFS와 완전 독립** — 버블 위치/크기/형태에 의존하지 않음
2. **인쇄 편차에 강건** — 균등 분할이므로 특정 열이 진한 것도 상대 비교로 감지
3. **계산이 단순** — 이미 있는 ImageData에서 평균 밝기만 계산
4. **교차 검증** — 틀린 감지를 사후에 잡아냄

## 한계

1. **ROI 박스가 정확해야** — 박스가 버블 영역을 정확히 감싸야 균등 분할이 의미있음
2. **인쇄된 번호/텍스트** — 버블 안에 번호가 인쇄되어 있으면 그것도 검은색으로 잡힘 → 중앙 영역만 샘플링으로 완화 가능
3. **불균등 버블 간격** — 실제 OMR이 완벽한 균등 배치가 아닐 수 있음 → 오차 마진 필요

## 이미지 소스

줌 팝업에서 보이는 것과 동일한 **ROI 박스 영역의 imageData**를 사용한다.
이미 `analyzeROI(imageData, ...)` 에 전달되는 데이터가 바로 이것이므로,
별도로 이미지를 가져올 필요 없이 같은 imageData를 그대로 균등 분할하면 된다.

## 구현 위치

```
js/omrEngine.js
  └── analyzeROI() 끝부분에 추가
      └── _overlayVerify(imageData, width, height, numQ, numC, isVert, bubbleResults)
          ├── 1) 균등 그리드 생성
          ├── 2) 구간별 darkRatio 계산
          ├── 3) 행별 overlayAnswer 산출
          └── 4) bubbleResults와 교차 검증 → row.overlayMatch 플래그 설정

js/canvas.js render()
  └── overlayMatch 플래그에 따라 셀 색상 변경
      ├── true → 기존 색상 유지 (초록/빨강)
      ├── false → 별색 (주황 테두리)
      └── null → 회색 (불확실)

js/ui.js updateRightPanel()
  └── 결과 그리드에 별색 클래스 추가
```

## 구현 가능 여부

**✅ 즉시 구현 가능**

- 필요한 데이터: `imageData` (이미 analyzeROI에 전달됨)
- 필요한 파라미터: `numQ`, `numC`, `orientation` (이미 있음)
- 새 함수 1개 추가 (`_overlayVerify`)
- UI 변경 최소 (row에 플래그 추가 → 기존 렌더링에 클래스만 추가)
- 기존 코드 변경 없음 — 분석 파이프라인 끝에 검증 단계만 append

**예상 작업량**: 함수 1개(~80줄) + UI 표시(~20줄) + 캔버스 오버레이(~15줄)
