# 채점 탭 기획서

## 개요
우측 사이드패널 또는 별도 탭으로 채점 결과를 3가지 표 + 시각화 자료로 제공.
모든 표는 CSV 다운로드 가능.

---

## 사전 조건: ROI 타입에 "기타" 추가

### 기타 영역
- ROI 타입에 `etc` (기타) 추가
- 기타 영역의 이름이 성적일람표의 추가 열 헤더가 됨
- 예: "한자급수" 라는 기타 영역 → 성적일람표에 "한자급수" 열 추가
- 기타 영역의 감지 결과(마킹값)가 해당 열의 값으로 들어감

---

## 1. OMR 결과표

### 구조
```
A: 응시번호
B: 성명
C: 점수
D~AQ: 마킹내용 (1번~40번, 디폴트 40문항, 실제 문항수에 맞춤)
AR~BU: 정오표 (1번~40번, O/X)
```

### 행 구성
- 1행: 헤더
- 2행~: 학생별 데이터 (이미지 순서)

### 데이터 소스
| 열 | 소스 |
|---|---|
| 응시번호 | exam_no ROI 감지값 또는 시험인원 매칭 |
| 성명 | 시험인원에서 매칭된 이름 (없으면 빈칸) |
| 점수 | gradeResult.totalScore |
| 마킹내용 N번 | results.rows[N-1].markedAnswer → choiceLabels[markedAnswer-1] |
| 정오 N번 | gradeResult.details[N-1].isCorrect → "O" / "X" |

### CSV 다운로드
- 파일명: `OMR결과표_[세션명]_[날짜].csv`
- UTF-8 BOM 포함 (엑셀 호환)

## 2. 성적일람표

### 기본 열 (디폴트)
```
A: 응시번호          ← exam_no ROI
B: 성명              ← 시험인원 매칭
C: 생년월일          ← birthday ROI 또는 시험인원
D: 수험번호          ← exam_no ROI
```

### 기타 항목 열 (사용자 추가)
```
E~: 기타 영역 이름들  ← etc 타입 ROI의 name이 헤더, 감지값이 데이터
```

### 성적 열 (디폴트, 토글 가능)
```
맞은개수      ← gradeResult.correctCount
점수          ← gradeResult.totalScore
표준점수      ← (개인점수 - 평균) / 표준편차 × 20 + 100
              ※ 표준점수 = ((X - μ) / σ) × 20 + 100
              ※ μ = 전체 평균 점수
              ※ σ = 전체 표준편차
석차          ← 점수 내림차순 순위 (동점자 동석차, 다음 순위 건너뜀)
              ※ 1등이 3명이면 다음은 4등
백분위        ← ((전체인원 - 석차) / 전체인원) × 100
              ※ 백분위 = ((N - R) / N) × 100
              ※ N = 전체 인원, R = 석차
```

### 열 토글
- 각 성적 열에 체크박스/토글 → 체크 해제하면 해당 열 숨김
- 기타 항목도 개별 토글 가능
- 토글 상태는 CSV 다운로드에도 반영 (숨긴 열은 CSV에서도 제외)

### CSV 다운로드
- 파일명: `성적일람표_[세션명]_[날짜].csv`

## 3. 문항분석표

### 3-1. 상/중/하위 그룹별 정답분석

#### 그룹 분류 (총점 기준)
```
전체 학생을 총점 내림차순 정렬 후:
- 상위 27%: 상위 ceil(N × 0.27)명
  ※ 27% 기준: 교육측정 표준 (Kelley, 1939)
- 하위 27%: 하위 ceil(N × 0.27)명
- 중위 46%: 나머지 (100% - 27% - 27%)
```

#### 표 구조
```
문항번호 | 정답 | 상위27% 정답수 | 상위27% 오답수 | 상위27% 계
                | 중위46% 정답수 | 중위46% 오답수 | 중위46% 계
                | 하위27% 정답수 | 하위27% 오답수 | 하위27% 계
```

### 3-2. 전체 정답률

```
문항별 정답률 = 해당 문항 정답자 수 / 전체 응시자 수 × 100

※ 정답률(%) = (정답자 수 / N) × 100
※ N = 전체 응시자 수
※ 난이도 해석:
   80%+ = 매우 쉬움
   60~80% = 쉬움
   40~60% = 보통
   20~40% = 어려움
   20%- = 매우 어려움
```

### 3-3. 전체 변별도

```
변별도 = (상부27% 정답자수 - 하부27% 정답자수) / (0.27 × 응답자수)

※ D = (U - L) / (0.27 × N)
※ U = 상위 27% 그룹에서 해당 문항 정답자 수
※ L = 하위 27% 그룹에서 해당 문항 정답자 수
※ N = 전체 응답자 수
※ 변별도 해석:
   0.40+ = 매우 좋음
   0.30~0.39 = 좋음
   0.20~0.29 = 보통 (개선 필요)
   0.10~0.19 = 나쁨 (수정 필요)
   0.10- = 매우 나쁨 (폐기 검토)
※ 음수 = 하위집단이 더 많이 맞춤 (문항 결함 의심)
```

### 3-3b. 정답률 차이 (참고 지표)

```
정답률 차이 = 상위집단 정답률 - 하위집단 정답률

※ ΔP = P_상위 - P_하위
※ P_상위 = U / upperN
※ P_하위 = L / lowerN
※ 변별도와 유사하나 비율 기반 비교
```

### 3-4. 문항응답분포 (상위 50% / 하위 50%)

```
상위 50%: 총점 상위 절반
하위 50%: 총점 하위 절반

각 문항별로:
- 1번 선택 수, 2번 선택 수, 3번 선택 수, 4번 선택 수, 5번 선택 수
- 미기입 수

※ 정답 선택지는 볼드 또는 색상 표시
※ 매력적 오답 분석: 오답 중 가장 많이 선택된 선택지 = 매력적 오답
```

#### 표 구조
```
문항 | 정답 | [상위50%] 1번 2번 3번 4번 5번 미기입 | [하위50%] 1번 2번 3번 4번 5번 미기입
  1  |  3  |          2   5  *20  1   2   0     |          8   6  *5   5   4   2
  2  |  1  |         *25  3   1   0   1   0     |         *10  5   6   3   4   2
```

### CSV 다운로드
- 파일명: `문항분석표_[세션명]_[날짜].csv`

## 4. 시각화 자료

### 4-1. 점수 분포 히스토그램
- X축: 점수 구간 (0~10, 11~20, ..., 91~100)
- Y축: 학생 수
- 평균선 표시
- Canvas 또는 SVG로 렌더링

### 4-2. 문항별 정답률 막대 그래프
- X축: 문항 번호 (1~N)
- Y축: 정답률 (0~100%)
- 40% 미만 문항 빨간색, 80% 이상 초록색
- 변별도가 음수인 문항 경고 표시

### 4-3. 문항-변별도 산점도
- X축: 정답률 (난이도)
- Y축: 변별도
- 이상적 문항: 정답률 40~60%, 변별도 0.3+
- 영역 구분선 표시

### 4-4. 선택지 분포 차트 (문항별)
- 특정 문항 클릭 시 표시
- 파이 차트 또는 막대: 각 선택지별 선택 비율
- 상위/하위 비교 (나란히 2개)

### 4-5. 석차-점수 산점도
- X축: 석차
- Y축: 점수
- 정규분포 곡선 오버레이

## 5. 계산식 요약

```javascript
// 평균
mean = Σ(scores) / N

// 표준편차
σ = √(Σ(score_i - mean)² / N)

// 표준점수 (T점수 변환, 평균 100, 표준편차 20)
T = ((X - mean) / σ) × 20 + 100

// 석차 (동석차 처리)
scores.sort(desc)
rank[i] = scores에서 score[i]보다 큰 값의 개수 + 1

// 백분위
percentile = ((N - rank) / N) × 100

// 정답률
correctRate = correctCount / N × 100

// 변별도
D = (U - L) / (0.27 * N)
// U = 상위27% 정답자수, L = 하위27% 정답자수, N = 전체 응답자수

// 정답률 차이 (참고)
deltaP = (U / upperN) - (L / lowerN)

// 상위/하위 27% 인원 수
upperN = ceil(N × 0.27)
lowerN = ceil(N × 0.27)

// 상위/하위 50% 인원 수
upperHalf = ceil(N / 2)
lowerHalf = N - upperHalf
```

## 6. UI 배치

### 탭 구조 (우측 사이드패널 또는 별도 영역)
```
[분석 결과] [OMR 결과표] [성적일람표] [문항분석표]
```

각 탭:
- 상단: 요약 통계 (평균, 최고점, 최저점, 응시인원)
- 중앙: 표 (스크롤)
- 하단: [CSV 다운로드] 버튼

문항분석표 탭:
- 상단: 요약 (평균 정답률, 평균 변별도)
- 중앙: 표 + 그래프 영역
- 하단: [CSV 다운로드]

## 7. 구현 순서

```
Step 1: ROI 타입에 "기타" 추가
Step 2: OMR 결과표 (가장 단순)
Step 3: 성적일람표 (표준점수/석차/백분위 계산)
Step 4: 문항분석표 (상/중/하위 그룹, 정답률, 변별도)
Step 5: 시각화 (히스토그램, 막대그래프, 산점도)
Step 6: CSV 다운로드 통합
Step 7: 열 토글/커스터마이즈
```
