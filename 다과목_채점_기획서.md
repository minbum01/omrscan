# 다과목 채점 기획서

## 현재 상태
- OMR 결과표/성적일람표/문항분석표가 **전체를 하나의 과목**으로 취급
- 과목 구분 없이 모든 영역의 답안을 하나로 합쳐서 표시
- 한 과목만 있는 시험에서는 문제없지만, 다과목이면 섞임

---

## 케이스 정리

### 케이스 1: 한 OMR 용지에 여러 과목

**예시**: 공무원 시험 — OMR 1장에 국어(1-25번) + 영어(26-50번) + 한국사(51-75번)

```
[OMR 1장]
├── 영역 "국어"   startNum=1   numQ=25
├── 영역 "영어"   startNum=26  numQ=25
└── 영역 "한국사" startNum=51  numQ=25

각 영역의 이름 = 과목명
```

**현재 문제**:
- 결과표에 1-75번이 한 줄로 나옴 (과목 구분 없음)
- 성적일람표 점수가 3과목 합산 점수 1개만 나옴
- 문항분석표도 1-75번 통합 — 과목별 난이도/변별도 분석 불가

**필요한 것**:
- OMR 결과표: 과목별로 섹션 구분 또는 과목 탭 선택
- 성적일람표: 과목별 점수 + 총점 (국어 80 / 영어 92 / 한국사 76 / 총점 248)
- 문항분석표: 과목 선택하여 해당 과목만 분석

---

### 케이스 2: 각 OMR 용지가 1과목씩, 같은 학생이 여러 OMR 제출

**예시**: 로스쿨 시험 — 언어이해 OMR 따로, 추리논증 OMR 따로

```
[이미지 세트 A: 언어이해]
├── 학생1_언어이해.jpg → 언어이해 80점
├── 학생2_언어이해.jpg → 언어이해 92점
└── 학생3_언어이해.jpg → 언어이해 76점

[이미지 세트 B: 추리논증]
├── 학생1_추리논증.jpg → 추리논증 88점
├── 학생2_추리논증.jpg → 추리논증 74점
└── 학생3_추리논증.jpg → 추리논증 90점
```

**필요한 것**:
- 학생1의 언어이해(80) + 추리논증(88) = 총점 168
- 같은 학생의 서로 다른 OMR을 **수험번호/이름으로 매칭**
- 성적일람표: 과목별 점수 열 + 총점 열

**현재 문제**:
- 이미지 1장 = 1행으로 처리됨
- 같은 학생의 2장이 별도 행으로 나옴 (합산 안 됨)

---

## 해결 방향

### 공통: "과목" 인식

현재 ROI의 `settings.name`이 과목명 역할을 하고 있음.
이걸 기반으로 과목별 데이터를 분리할 수 있음.

```
학생1의 데이터:
  국어: { score: 80, answers: [...], correctCount: 20 }
  영어: { score: 92, answers: [...], correctCount: 23 }
  한국사: { score: 76, answers: [...], correctCount: 19 }
  총점: 248
```

### 케이스 1 구현 방향

**collectData에서 과목별 분리**:
```javascript
row = {
    examNo: '20260001',
    name: '홍길동',
    subjects: {
        '국어': { score: 80, answers: [...], correctCount: 20 },
        '영어': { score: 92, answers: [...], correctCount: 23 },
    },
    totalScore: 172,
}
```

**OMR 결과표**: 과목 선택 드롭다운 → 선택한 과목의 마킹/정오만 표시
**성적일람표**: 과목별 점수 열 자동 생성 (국어 / 영어 / 총점)
**문항분석표**: 과목 선택 드롭다운 → 해당 과목만 분석

### 케이스 2 구현 방향

**같은 학생 합산**:
```
이미지A(학생1, 언어이해) + 이미지B(학생1, 추리논증)
→ 수험번호 매칭 → 하나의 행으로 합산
```

수험번호(또는 이름/전화번호)가 같으면 같은 학생으로 판단하여 과목별 점수를 합침.

```javascript
// 합산 후
row = {
    examNo: '20260001',
    name: '학생1',
    subjects: {
        '언어이해': { score: 80, ... },  // 이미지A에서
        '추리논증': { score: 88, ... },  // 이미지B에서
    },
    totalScore: 168,
}
```

## 피드백 필요

### Q1. 과목이 여러 개일 때 OMR 결과표를 어떻게 보여줄까?
- (A) 과목별 탭으로 전환 (한 번에 1과목만)
- (B) 과목 전체를 가로로 이어붙임 (국어 1-25번 | 영어 26-50번 | ...)
- (C) 과목 선택 드롭다운 (기본: 전체, 선택 시 해당 과목만)

### Q2. 성적일람표에서 과목별 점수 열을 어떻게?
- (A) 자동으로 과목 수만큼 열 추가 (국어 / 영어 / 한국사 / 총점)
- (B) 뱃지로 추가/제거 가능 (기존 시스템과 동일)
- (C) 둘 다 (자동 생성 + 뱃지로 커스터마이징)

### Q3. 케이스 2에서 같은 학생 판별 기준
- 수험번호가 최우선? 이름? 전화번호?
- 시험인원 데이터에 등록된 정보 기준?
- 매칭 실패 시 어떻게? (별도 행으로 유지? 경고?)

### Q4. 케이스 2에서 이미지를 어떻게 구분?
- 같은 세션에 언어이해 + 추리논증 이미지를 같이 올리는 건가?
- 아니면 세션을 분리해서 올리고, 나중에 합치는 건가?
- 이미지별로 "이 이미지는 어떤 과목인지" 사용자가 지정?

### Q5. 문항분석표 — 과목별 분석?
- 과목 선택 드롭다운으로 해당 과목만 분석
- 전체 선택 시 모든 문항 통합 분석
- 과목별 난이도/변별도가 따로 나와야 의미 있음
