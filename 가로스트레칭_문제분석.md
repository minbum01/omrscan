# 가로 스트레칭 분석 문제 — 상세 설명

## 요약
이미지를 가로로 늘려서 길쭉 버블을 동그랗게 만들면 BFS 감지가 작동할 것 같았지만, **절반만 감지**되고 나머지는 탈락합니다. 마킹 판별도 불안정합니다. 이 문서는 **왜 그런지**를 로그를 기반으로 단계별로 설명합니다.

---

## BFS가 뭘 찾는지 먼저 이해하기

### 빈 버블과 꽉 찬 버블의 차이
```
  빈 버블 (unmarked)       꽉 찬 버블 (marked)
  ┌─────────┐              ┌─────────┐
  │ ○○○○○○  │              │ ●●●●●●  │
  │○  ①  ○ │              │●████████│
  │○     ○  │              │█████████│
  │ ○○○○○○ │              │ ●●●●●●  │
  └─────────┘              └─────────┘
  테두리만 어두움           전체가 어두움
  내부는 흰색 (숫자 제외)   
```

### BFS 필터의 3가지 기준
```
1. 종횡비(aspect ratio): 0.7 ~ 1.4 (거의 정사각형/원형)
2. 채움률(fill ratio): 0.3 ~ 0.85 (이 범위만 "빈 버블"로 인정)
3. 크기: 중앙값 대비 30% ~ 180%
```

### 채움률(fill)이 뭐냐
`fill = (BFS로 찾은 어두운 픽셀 수) ÷ (블롭 바운딩박스 면적)`

- 빈 버블: 테두리만 어두움 → `fill ≈ 0.25 ~ 0.45` (얇은 링)
- 꽉 찬 버블: 전체 어두움 → `fill ≈ 0.85 ~ 1.00`
- **중간값(0.3~0.85)만 빈 버블로 인정**. 나머지는 탈락.

---

## 로그 분석

### 통과한 블롭 26개 — 두 그룹이 있음

**그룹 A: "꽉 찬 버블" (마킹된 것)**
```
블롭1:  35x33  fill=0.85  ← 상한선 근처
블롭2:  35x28  fill=0.84
블롭3:  35x28  fill=0.87  ← 0.85 초과 직전
블롭4:  39x28  fill=0.83
블롭5:  37x29  fill=0.81
블롭13: 36x29  fill=0.85
블롭23: 36x32  fill=0.82
```
- fill이 0.81~0.87로 **"꽉 찬 버블" 판정 경계**에 있음
- 본래 이 필터는 "빈 버블만" 찾는 건데, 꽉 찬 버블이 같이 통과됨
- `fill < 0.85` 상한을 간신히 넘겨서 통과

**그룹 B: "빈 버블" (비어있는 것)**
```
블롭6:  29x23  fill=0.35
블롭7:  30x23  fill=0.34
블롭10: 30x24  fill=0.35
...
```
- fill 0.30~0.35
- 하한 `fill > 0.3`을 간신히 통과

### 탈락한 블롭 — 핵심 문제

**대다수가 "채움률↓" 탈락**
```
❌ 28x23 fill=0.23 (채움률↓)
❌ 28x23 fill=0.25 (채움률↓)
❌ 29x24 fill=0.21 (채움률↓)
❌ 28x23 fill=0.22 (채움률↓)
... (수십 개)
```
- fill 0.21~0.26 → `0.3` 하한 미달로 탈락
- **이게 진짜 빈 버블인데 탈락**함

---

## 왜 빈 버블의 fill이 0.3 미만으로 떨어졌나?

### 핵심 원인: 스트레칭 + 보간 = 테두리 흐릿해짐

**원본 버블:**
```
 x x x x x
x . . . . x    테두리 픽셀 값 = 검정(0)
x . ① . . x   내부 픽셀 값 = 흰색(255)
x . . . . x
 x x x x x

예: 20x10 크기
테두리 어두운 픽셀 ≈ 50개
면적 = 200
fill = 50 / 200 = 0.25
```

잠깐 — 원본이 이미 0.25면 탈락해야 정상인데 왜 어떤 건 통과하고 어떤 건 탈락할까?

답: **원본 버블의 테두리 두께가 일정하지 않음.** 스캔 품질 편차, 인쇄 편차, 이미지 압축 아티팩트 때문에 어떤 버블은 테두리가 선명하고 어떤 버블은 흐릿함.

### 스트레칭 후 상황

**2배 가로 스트레칭 + 이중선형 보간:**
```
x x x x x x x x x x
x . . . . . . . . x    테두리 픽셀이 늘어나면서
x . ① . . . . . x     중간값(회색)이 생김
x . . . . . . . x     원래 검정이었던 픽셀이
 x x x x x x x x x    어두운 회색으로 바뀜

40x10 크기 (가로만 2배)
테두리 어두운 픽셀 ≈ 60개 (테두리 수는 비슷)
면적 = 400 (2배)
fill = 60 / 400 = 0.15
```

**→ 가로만 늘리면 어두운 픽셀 수는 거의 그대로인데 면적만 커짐 → fill 비율 급감**

게다가 **이중선형 보간**으로 테두리 픽셀이 회색으로 변해서, `darkThreshold` 미만으로 떨어지면 BFS가 "어두움"으로 안 셈.

결과: 빈 버블의 fill이 `0.25` → `0.15~0.22`로 떨어짐 → **0.3 하한 미달 → 탈락**

## 마킹 판별도 불안정

Stage 2 로그:
```
Q1: [1] s=253 [2] s=396 [3] s=417 [4] s=5 [5] s=458 → 5
Q2: [1] s=336 [2] s=253 [3] s=418 [4] s=4 [5] s=450 → 1
Q3: [1] s=287 [2] s=1338 [3] s=415 [4] s=4 [5] s=420 → 2 ← 강한 판정
Q4: [1] s=345 [2] s=412 [3] s=433 [4] s=6 [5] s=450 → 1
Q5: [1] s=311 [2] s=442 [3] s=253 [4] s=5 [5] s=477 → null
```

### 왜 Q3만 정답이 확실하고 나머지는 애매한가?

**Q3의 [2]번**: `score=1338, fill=1.000`
- 이건 **진짜 꽉 찬 마킹**. 블롭이 통째로 까만 덩어리.
- `prom=999` (압도적)

**Q1, Q2, Q4, Q5**: 점수가 300~480 범위에 몰려있음
- 점수 차이가 작음 (prominence 1.16~1.36)
- 표준 마킹 기준 `prom > 1.5`를 못 넘음
- `fill` 값도 0.28~0.39로 애매함

### 원인: 스트레칭이 마킹 강도를 희석시킴
- 꽉 찬 마킹이 가로로 늘어나면서 픽셀 밝기가 평균화됨
- 이중선형 보간으로 중간값이 생김
- "완전히 까만 영역"이 줄어듦
- 빈 버블과 마킹된 버블의 **대비(contrast)가 낮아짐**
- → 어느 게 마킹인지 애매해짐

## 정리: 스트레칭의 근본적 한계

| 관점 | 문제 |
|------|------|
| **채움률** | 가로로만 늘리면 어두운 픽셀 수는 그대로인데 면적만 커짐 → fill 비율 떨어짐 |
| **보간 흐림** | 이중선형 보간이 테두리 선명도를 떨어뜨려서 어두운 픽셀 수마저 줄어듦 |
| **마킹 대비** | 꽉 찬 마킹도 희석돼서 빈 버블과 구분이 어려워짐 |
| **해상도** | 원본이 작으면 작을수록 보간 손실이 큼 |

### 핵심 한 줄
> **가로 스트레칭은 BFS의 "종횡비" 문제는 해결하지만, "채움률" 계산을 망가뜨린다.**
>
> BFS 필터는 원형 버블 + 선명한 테두리 + 픽셀 밀도 균형을 전제로 설계됐는데, 스트레칭은 이 셋 중 둘(테두리 선명도, 픽셀 밀도)을 깨뜨림.

## 해결 방향 후보 (구현 안 함)

1. **채움률 필터를 스트레칭 비율에 따라 자동 조정**
   - `stretchRatio=2`면 `fill` 하한을 `0.3 → 0.15`로 낮춤
   - 단점: 노이즈도 같이 통과할 수 있음

2. **보간 방식을 nearest-neighbor로 변경**
   - 테두리가 각지게 유지됨
   - 대신 버블 모양이 울퉁불퉁해짐

3. **스트레칭 후 샤프닝 필터 적용**
   - 테두리 선명도 복원
   - 복잡도 증가

4. **스트레칭 대신 BFS의 종횡비 필터를 길쭉 버블용으로 따로 만들기**
   - 종횡비 0.35~1.4로 확장 + 채움률은 원본 그대로
   - 가장 안전하지만 노이즈 구분 기준이 달라짐

5. **완전히 다른 감지 방식**
   - 프로젝션, 템플릿 매칭, 캘리브레이션 등 (이전에 논의한 방안들)

---

## 결론
가로 스트레칭은 **개념적으로는 맞지만**, BFS 엔진이 기대하는 전제(선명한 테두리, 픽셀 밀도 균형)를 깨뜨려서 실용적으로는 부작용이 큽니다. 특히 저해상도/저품질 이미지에서 심합니다.
