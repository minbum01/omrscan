# 버그 수정 및 기능 개선 구현 계획

버그확인.txt + 사용자 답변 반영 최종 버전. 각 Step은 독립적으로 커밋 가능한 단위로 설계됨.

## Step 1. ROI 설정 팝업 - 배경 클릭으로 닫히지 않도록

**문제**: 박스 그린 후 설정 팝업 뜰 때 바깥 클릭하면 팝업이 사라짐.

**수정**: `js/ui.js` `openRoiSettingsPopup()` 의 background click 리스너 제거.
- **닫는 방법**: 취소 버튼, 확인 버튼, ESC 키만 허용
- ESC 키 처리는 `js/shortcuts.js`에 `#roi-settings-popup` 모달이 열려있으면 취소 처리

**파일**: `js/ui.js`, `js/shortcuts.js`

## Step 2. numQuestions 0 버그 수정

**문제**: 사이드패널에서 문항수가 0으로 표시되고 무한대로 분석 돌아감.

**원인**: `applyRoiSettingsPopup()`와 `setOrientation()`에서 방향 변경 시 `numQuestions = 0`으로 리셋. 이후 자동감지에 의존하는데 실패하면 0인 채로 남음.

**수정**:
- 방향 변경 시 0으로 리셋하지 말고, 기존 값 유지
- 자동감지는 **초기 박스 생성 시에만** 동작
- 사용자가 이미 입력한 값은 덮어쓰지 않음

**파일**: `js/ui.js`

## Step 3. 과목 시스템 (과목 코드 우선)

### 사용자 요구사항
- **과목 코드 기반이 우선** — OMR에 과목 코드 영역이 있으면 자동 매칭
- **과목 코드 없으면 사용자 수동 입력** — 이름 기반으로 묶기
- **한 번 작성하면 재사용** — 전역 저장, 다음에 불러옴
- **휴먼 오류 최소화** — 동일 이름은 자동으로 같은 과목으로 인식

### 데이터 구조
```javascript
// App.state.subjects (전역, 한 번 만들면 영구 저장/로드)
subjects: [
    {
        code: '1',           // 과목 코드 (없으면 빈 문자열)
        name: '영어',        // 과목 이름
        numQuestions: 40,    // 문항 수
        numChoices: 5,       // 선택지 수
        answers: '1243525...', // 정답 문자열
        scorePerQuestion: 2.5
    },
    ...
]

// roi.settings 추가 필드
subjectName: '영어',  // 이 ROI가 속한 과목 (이름으로 매칭)
```

### 워크플로우
1. **과목 코드 영역이 있는 경우**:
   - `subject_code` 타입 영역 설정 → codeList에 {코드, 과목명} 매핑
   - 답안 영역은 `answerSource: 'code_N'`으로 해당 과목 코드 영역 참조
   - 분석 시 코드 자동 감지 → 과목 매칭 → 정답 적용
2. **과목 코드 없는 경우**:
   - 답안 영역에서 **"과목 선택" 드롭다운** 사용
   - 기존 등록된 과목 선택 or "새 과목 추가"
   - 선택한 과목의 정답/배점이 자동 적용
3. **과목 관리 UI**:
   - 상단 패널에 "과목 관리" 버튼 (기존에 있으면 개선)
   - 과목 추가/수정/삭제
   - 저장된 과목은 세션 간 유지 (localStorage)

### 파일
`js/ui.js`, `js/grading.js`, `js/app.js`, `js/subjectManager.js`

## Step 4. 선택지 직접 입력 전면 개편 (프리셋 완전 제거)

### 사용자 요구사항
- **프리셋 완전 제거** — 특수문자, 단어 형태 등 모든 것 사용자 직접 입력
- 선택지 개수 입력 → 해당 개수만큼 텍스트박스 생성

### UI 흐름
```
[선택지 개수] [5]
[선택지] [1] [2] [3] [4] [5]    ← 개수만큼 텍스트박스 자동 생성
```
- `numChoices` 값 변경 시 텍스트박스 개수가 즉시 재생성됨
- 각 텍스트박스는 `choiceLabels[i]` 값
- 사용자가 자유롭게 수정 (예: "ㄱ", "옳음", "참", "①", 긴 단어 등 모두 가능)
- **CHOICE_PRESETS 객체 완전 삭제**
- `choicePreset`, `customLabels` 필드도 삭제

### 채점 로직 변경
- 기존: `parseInt(ch)` 로 숫자만 파싱
- 변경: `choiceLabels[i]` 배열 인덱스 기반으로 매칭
- 정답 입력도 라벨 기반 (예: "ㄱ,ㄴ,ㄷ,...")

### 파일
`js/ui.js`, `js/grading.js`

## Step 5. 버블 감지 슬라이더 (접기/펼치기 + 기본값 자동 전환)

### 사용자 요구사항
- 슬라이더는 **평소엔 접혀있음**, 클릭 시 펼쳐짐
- **길쭉 버블 체크박스에 따라 기본값 자동 전환**
  - 동그란 버블: `h/w 하한 = 0.7` 같은 기존 원형 기본값
  - 길쭉 버블: `h/w 하한 = 1.4` 같은 길쭉용 기본값
- 필요할 때만 조정, 기본값이 제대로 로드되어야 함

### 구현 방식
1. **엔진은 분리 유지**: `_stage1_findBubbles` / `_stage1_findBubbles_elongated` 둘 다 존재
2. **설정 필드 하나로**: `minHW`, `maxHW`, `minFill` (prefix 없음, 공통)
3. **길쭉 모드 토글 시**:
   - 체크 OFF → 기본값 `minHW=0.7, maxHW=1.4, minFill=0.3`
   - 체크 ON → 기본값 `minHW=1.4, maxHW=5.0, minFill=0.15`
   - 토글 시 자동으로 해당 기본값으로 리셋
4. **UI 배치**:
   ```
   [☐ 길쭉 버블 분석]
   [▸ 버블 감지 설정]  ← 접혀있음, 클릭하면 펼쳐짐
     슬라이더들...
   ```
5. **팝업과 사이드바 양쪽에서 동일한 UI**

### 파일
`js/ui.js`, `js/omrEngine.js`, `js/canvas.js`

## Step 8. 회전 개별 적용 지원 + 조정바 UI 통일

### 사용자 요구사항
- 현재: 회전은 무조건 전체 적용
- 수정: 개별/전체 모두 가능하게

### 통일 UI 패턴 (사용자 승인)
```
[진하기 슬라이더] [□ 전체 적용]
[기울기 -] [기울기 +] [□ 전체 적용]
[좌회전] [우회전] [□ 전체 적용]
[원상복구]
```
- 모든 조정에 **동일한 "전체 적용" 체크박스** 패턴
- 체크 OFF: 현재 이미지만
- 체크 ON: 전체 이미지
- 기존 "전체 적용 버튼"은 제거 (체크박스로 통일)

## Step 9. [추가 버그] 영역 선택 + Delete 키 안 지워지는 문제

**사용자 제보**: 영역을 누르고 Delete를 눌러도 안 지워질 때가 있음.

**조사 방향**:
- `js/shortcuts.js`의 Delete 처리 로직 확인
- 포커스가 input/textarea에 있을 때는 무시되는 조건 확인
- `CanvasManager.selectedRoiIdx` 상태 동기화 확인
- 캔버스 클릭 시 `selectedRoiIdx` 설정이 빠진 경로가 있는지 확인

**파일**: `js/shortcuts.js`, `js/canvas.js`

## Step 10. [추가 버그] 영역 클릭 시 우측 사이드바 자동 스크롤 복구

**사용자 제보**: 이전에는 영역을 클릭하면 우측 사이드바에서 해당 영역 카드로 스크롤됐는데 지금은 작동 안함.

**조사 방향**:
- `CanvasManager.selectedRoiIdx` 변경 시 스크롤 트리거 확인
- `UI.updateRightPanel()` 호출 후 `scrollIntoView()` 누락 확인
- `.roi-card-selected` 클래스가 붙은 요소를 찾아서 스크롤하는 로직 복구

**파일**: `js/canvas.js`, `js/ui.js`

## Step 11. [신규 기능] 영역 목록 탭 이동

### 사용자 요구사항
현재: 우측 사이드바 **상단**에 영역 목록(순서 바꾸기)이 고정으로 표시 → 자리 많이 차지함.

변경 후:
```
┌────────────────────────────────────┐
│ [영역 추가] [영역 목록] [양식 불러오기] │ ← 탭 버튼들
├────────────────────────────────────┤
│                                    │
│  선택된 탭의 내용                    │
│                                    │
└────────────────────────────────────┘
```
- "영역 추가", "영역 목록", "양식 불러오기" 3개 버튼
- "영역 목록" 버튼 누르면 순서 변경 UI 펼쳐짐
- 평소엔 접혀있어서 자리 차지 안 함
- 기존 상단에 있던 영역 목록은 제거

### 파일
`js/ui.js`

## Step 12. 전체 점검 + 추가 버그 찾기

Step 1~11 완료 후:
- 통합 테스트: 템플릿 저장/로드, 배치 처리, 채점 결과
- 수동 테스트 항목:
  - 영역 그리기 → 설정 팝업 → ESC/취소/확인
  - 영역 변경(순서, 방향) 시 numQuestions 유지
  - 과목 매니저: 등록 → 저장/로드 → 과목코드 매칭 / 수동 매칭
  - 선택지 직접 입력 → 한글/특수문자/단어 OMR 시뮬레이션
  - 길쭉 버블 + 배치 + 슬라이더 기본값 자동 전환
  - 회전 개별/전체, 기울기 개별/전체, 진하기 개별/전체 (일관성)
  - Delete 키로 영역 삭제
  - 영역 클릭 → 사이드바 자동 스크롤
  - 영역 목록 탭 펼치기/접기
- 콘솔 에러, 예외 처리 누락 확인

## 구현 순서 (권장)

Step 7(제거) → 2(numQ 버그) → 1(팝업 닫힘) → 9(Delete) → 10(스크롤) → 11(탭 이동) → 4(선택지) → 5(슬라이더) → 3(과목) → 6(배치) → 8(조정바) → 12(점검)

**이유**:
1. 먼저 **제거(7)** 해서 코드 단순화
2. **작은 버그(2, 1, 9, 10)** 먼저 해결
3. **UI 재구성(11)** 으로 레이아웃 정리
4. **큰 기능(4, 5, 3)** 차례로 구현
5. **배치(6)** 는 5번 완료 후 자연스럽게 함께
6. **조정바 통일(8)** 은 독립적
7. 마지막에 **전체 점검(12)**

## 사용자 답변 요약

| 항목 | 답변 |
|------|------|
| 1. 과목 그룹핑 방식 | 과목 코드 우선, 없으면 수동, 한번 작성 재사용, 휴먼오류 최소화 |

## 구현 시 주의사항

1. **각 Step마다 독립 커밋** — 문제 생기면 개별 롤백 가능
2. **기존 동작 영향 최소화** — 특히 분석 엔진, 채점 로직은 신중히
3. **테스트 이미지 기반 검증** — 기존에 잘 되던 OMR은 계속 잘 되어야 함
4. **Step 7(제거) 먼저** — 중복 로직 줄이고 이후 작업 단순화
