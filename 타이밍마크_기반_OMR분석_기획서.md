# 타이밍 마크 기반 OMR 분석 기획서

## 개요
타이밍 마크를 이용하여 OMR 버블의 **열 위치를 자동 추출**하고, 사용자가 **행 위치를 직접 지정**하여 그리드를 구성한다. 기존 BFS 블롭 감지와 별개의 새로운 로직이다.

## 핵심 원칙
- **하드코딩 금지** — 모든 좌표는 타이밍 마크에서 실시간 계산
- 템플릿에는 **절대 좌표가 아닌 타이밍 마크 대비 상대 비율**만 저장
- 매 장 분석 시 해당 장의 타이밍 마크 실측값에 비율을 적용

---

## 타이밍 마크란
- OMR 용지 가장자리(상단/하단/좌측/우측)에 인쇄된 짧은 검은 막대
- 각 막대 하나가 한 열(또는 한 행)에 대응
- **각 변의 첫 번째 타이밍 마크는 앵커** — 위치 기준점 역할만 하며 데이터 열이 아님

---

## 워크플로우

### 1단계: 템플릿 생성 (최초 1회)

#### 1-1. 타이밍 마크 감지
- 이미지에서 보이는 타이밍 마크를 **모두** 감지 (상/하/좌/우 어디에 있든)
- 각 변의 첫 번째 마크 = **앵커** (기준점)

#### 1-2. 실선 표시 (전체 페이지)
- 상/하단 타이밍 마크 → **세로 실선** (빨간색, 전체 페이지 높이)
  - 상단과 하단 마크가 모두 있으면 → 1:1 대응시켜 상단~하단을 잇는 선 (휘어짐 보정)
  - 한쪽만 있으면 → 수직 직선
- 좌/우측 타이밍 마크 → **가로 실선** (빨간색, 전체 페이지 너비)
  - 좌측과 우측 마크가 모두 있으면 → 1:1 대응시켜 잇는 선
  - 한쪽만 있으면 → 수평 직선

#### 1-3. 영역 설정 (사용자 수동)
- 실선이 그어진 캔버스 위에서 **영역(ROI)을 드래그**
- 각 영역별로 설정:
  - **영역 타입**: 과목 답안, 수험번호, 생년월일 등
  - **인식 방향**: 세로형 / 가로형
  - **해당 영역에 포함되는 열 범위**: 세로선 중 몇 번째~몇 번째
  - **문항 수, 선택지 수, 시작 번호** 등

#### 1-4. 행 위치 지정 (사용자 수동, 영역별)
- 각 영역 내에서 사용자가 **행 위치를 하나씩 클릭**
- 클릭할 때마다 파란 가로선이 추가됨
- 행 위치는 **앵커 대비 상대 비율**로 저장 (절대 y좌표가 아님)
- 실수 시 되돌리기(undo) 가능

#### 1-5. 그리드 행렬 생성
- 세로선(열) × 파란 가로선(행) 교차점 = OMR 셀
- 영역별 그리드를 행렬화 → 채점의 기본 토대

#### 1-6. 템플릿 저장
- 타이밍 마크 전체 위치 (앵커 포함, 기준용)
- 영역별:
  - 열 인덱스 범위 (타이밍 마크 번호 기준)
  - 행 위치: **앵커 대비 상대 비율 배열**
  - 인식 방향, 타입, 문항 수, 선택지 수, 시작 번호

---

### 2단계: 분석 (매 장)

#### 2-1. 타이밍 마크 재감지
- 현재 스캔 이미지에서 타이밍 마크 추출
- 앵커 마크 식별

#### 2-2. 열 위치 계산 (장마다 실시간)
- 감지된 타이밍 마크의 **실제 위치를 그대로 열 좌표로 사용**
- 상/하단 마크가 모두 있으면 → 각 행 높이에서 상/하단 보간으로 열 위치 계산 (휘어짐 보정)

#### 2-3. 행 위치 계산 (비율 환산)
- 템플릿의 행 비율을 현재 이미지의 앵커 위치에 적용
- 예: 템플릿에서 행1 = 앵커로부터 15% 위치 → 현재 이미지 앵커 기준 15% 지점

#### 2-4. 셀 좌표 결정
- 각 셀의 (x, y) = 해당 행에서의 보간된 열 위치 × 환산된 행 위치
- 셀 샘플링 크기 = 인접 열 간격, 인접 행 간격에서 자동 계산

#### 2-5. 셀 샘플링 → 마킹 판별
- 기존 Stage 2 로직 (sampleCell → prominence 기반 마킹 판별) 그대로 사용

---

## 좌표 저장 방식

| 항목 | 저장 형태 | 매 장 적용 방식 |
|------|-----------|----------------|
| 열 위치 | 타이밍 마크 인덱스 (1번, 2번...) | 해당 장에서 감지된 마크의 실제 x좌표 사용 |
| 행 위치 | 앵커 대비 상대 비율 (0.0~1.0) | 해당 장의 앵커 위치에 비율 곱셈 |
| 셀 크기 | 저장 안 함 | 인접 열/행 간격에서 실시간 계산 |

## 휘어짐 보정

```
상단 마크 N번: x = 50        상단 마크 N번: x = 90
    \                             |
     \ (용지 휘어짐)              | (정상)
      \                           |
하단 마크 N번: x = 48        하단 마크 N번: x = 90
```

- 행 y에서의 열 N 위치 = lerp(상단x, 하단x, (y - 상단y) / (하단y - 상단y))
- 각 셀마다 독립적으로 보간 → 용지 전체가 휘어져도 정확

## 기존 코드와의 관계
- BFS 블롭 감지(`_stage1_findBubbles`)를 **사용하지 않음** — 완전 별도 로직
- Stage 2(셀 샘플링 → 마킹 판별)부터는 **기존 로직 공유**
- 기존 자동감지 모드는 그대로 유지 (타이밍 마크가 없는 일반 OMR용)
- 두 모드 공존: 사용자가 OMR 양식에 따라 선택
