# OMR 채점 시스템 기획서

## 1. 프로젝트 개요

### 1.1 목적
PC에서 스캔/촬영한 OMR 답안지 이미지를 업로드하여 자동으로 마킹을 인식하고, 정답과 대조하여 채점하는 웹 기반 시스템을 개발한다.

### 1.2 핵심 가치
- **실전 환경 대응**: 그림자, 조명 불균일, 기울어진 스캔 등 다양한 조건에서도 정확한 판독
- **다양한 필기구 지원**: 검정 사인펜, 빨간 색연필, 파란 볼펜 등 다양한 필기구로 마킹한 답안 인식
- **서버 불필요**: 브라우저 내장 Canvas API(무료)로 별도 서버 없이 PC 브라우저에서 완전 동작
- **자동 채점**: 정답 입력 후 자동 대조, 점수 산출, 결과 파일 내보내기까지 일괄 처리

### 1.3 동작 환경
- **PC 전용** (Windows / Chrome, Edge 등 모던 브라우저)
- 모바일 대응 불필요

## 2. 시스템 구성

### 2.1 기술 스택
| 구분 | 기술 | 비용 |
|------|------|------|
| 프론트엔드 | HTML5 Canvas, JavaScript | 무료 (브라우저 내장) |
| UI 프레임워크 | Tailwind CSS (CDN) | 무료 |
| 이미지 처리 | Canvas API (픽셀 단위 읽기/쓰기) | 무료 (브라우저 내장) |

## 3. 기능 상세

### 3.1 이미지 관리
| 기능 | 설명 |
|------|------|
| 다중 파일 업로드 | 여러 장의 OMR 이미지 동시 업로드 |
| 이미지 목록 | 좌측 사이드바에 파일명 표시, 클릭으로 전환 |
| 테스트 양식 생성 | 가상 OMR 양식(그림자+빨간펜 시뮬레이션)을 자동 생성하여 기능 테스트 |

### 3.2 캔버스 인터랙션
| 기능 | 설명 | 단축키 |
|------|------|--------|
| 화면 이동 모드 (Pan) | 이미지를 스크롤로 탐색 | `V` |
| 영역 박스 모드 (Draw) | 드래그로 분석 대상 ROI 지정 | `D` |
| 박스 하나 되돌리기 (Undo) | 마지막에 그린 ROI 박스 1개 삭제 | `Ctrl+Z` |
| 박스 전체 지우기 | 현재 이미지의 모든 ROI 초기화 | `Ctrl+Shift+Z` |
| 확대 | 캔버스 줌 인 | `Ctrl++` 또는 마우스 휠 |
| 축소 | 캔버스 줌 아웃 | `Ctrl+-` 또는 마우스 휠 |
| 화면 맞춤 | 캔버스를 화면 크기에 맞춤 | `Ctrl+0` |
| 분석 실행 | 현재 이미지 정밀 분석 | `Enter` |

### 3.3 정답 입력 및 채점 (신규)
| 기능 | 설명 |
|------|------|
| 정답 입력 UI | 모달 또는 패널에서 문항 수, 지선다 수 설정 후 각 문항 정답 번호 입력 |
| 문항별 배점 설정 | 기본 균등 배점, 필요시 문항별 개별 배점 지정 가능 |
| 자동 채점 | 판독된 마킹과 정답을 자동 대조하여 맞음(✅)/틀림(❌) 판정 |
| 점수 산출 | 맞은 문항의 배점 합산으로 총점 계산 |
| 오답 표시 | 결과 패널에서 틀린 문항 강조, 캔버스에서도 틀린 버블 색상 구분 |

### 3.4 일괄 처리 (신규)
| 기능 | 설명 |
|------|------|
| 일괄 분석 | 업로드된 모든 이미지에 대해 동일한 ROI 설정으로 연속 분석 |
| 진행 표시 | 프로그레스 바로 현재 처리 상태 표시 (3/10장 완료 등) |
| 일괄 채점 | 입력된 정답으로 전체 답안지 한번에 채점 |

### 3.5 결과 내보내기 (신규)
| 기능 | 설명 |
|------|------|
| CSV 다운로드 | 파일명, 문항별 마킹값, 정오 여부, 총점을 CSV 파일로 다운로드 |
| Excel 다운로드 | 동일 데이터를 .xlsx 형식으로 다운로드 |

### 3.6 OMR 판독 엔진 (reference.html 기반)

#### 3.6.1 전처리 - 그림자/색연필 대응 이진화
```
원본 픽셀 → Min(R,G,B) 추출 → 영역별 동적 임계값 계산 → 이진화
```
- **Min(R,G,B)**: 빨간펜도 어두운 픽셀로 변환하여 색연필 마킹 인식
- **동적 임계값**: 영역 평균 밝기의 85%를 기준으로 그림자 환경 대응

#### 3.6.2 블롭 탐지 - BFS 군집화
```
이진화 픽셀 → 4방향 BFS → 연결 픽셀 군집(Blob) → 바운딩박스 + 잉크밀집도 계산
```

#### 3.6.3 필터링 - 노이즈 제거
| 단계 | 조건 |
|------|------|
| 1차: 크기/형태 | 8~80px, 가로세로비 0.4~2.5 |
| 2차: 중간값 기반 | 면적이 중간값의 50%~150% 범위 내 |

#### 3.6.4 구조화 - 행/열 분석
```
블롭 Y좌표 정렬 → 행 그룹핑(허용 오차: 버블 높이 75%) → X좌표 정렬
```

#### 3.6.5 마킹 판별 - 상대적 비교
```
각 행 내:
  점수 = (잉크밀집도 × 500) + (255 - 평균밝기)
  → 최고 점수 블롭 선정
  → 행 평균 대비 밝기차 > 8 또는 잉크밀집도 차 > 0.03 이면 마킹 확정
  → 차이 미미하면 "미기입" 처리
```

### 3.7 결과 표시
| 기능 | 설명 |
|------|------|
| 캔버스 오버레이 | 인식 버블 = 초록 원, 정답 마킹 = 파란 원, 오답 마킹 = 빨간 원 |
| 결과 패널 | 문항별 정답/마킹값/정오 표시 + 총점 |
| 미기입 경고 | 마킹 미인식 문항 빨간색 강조 |

## 4. 사용자 플로우

```
1. OMR 답안지 이미지 업로드 (스캔 또는 촬영)
   ↓
2. 정답 입력 (문항 수, 지선다 수, 각 문항 정답 번호, 배점)
   ↓
3. 좌측 목록에서 이미지 선택
   ↓
4. "영역 박스 치기" 모드(D키)로 전환 → ROI 드래그 지정
   ↓
5. "정밀 분석 실행" (Enter키) → 마킹 인식
   ↓
6. 자동 채점 → 결과 패널에서 문항별 정오 + 총점 확인
   ↓
7. [선택] 일괄 채점: 동일 ROI로 나머지 답안지 연속 처리
   ↓
8. CSV/Excel 다운로드로 결과 저장
```

## 5. reference.html 대비 변경 사항

### 5.1 제거 항목 (모바일 관련)
- 터치 이벤트 코드 (`touchstart`, `touchmove`, `touchend`) 제거
- 모바일 반응형 분기 (`md:` 클래스) 제거 → PC 고정 레이아웃
- `user-scalable=no`, `overscroll-behavior` 등 모바일 설정 제거
- 모바일 스크롤 안내 UI 제거

### 5.2 추가 항목
| 기능 | 상세 |
|------|------|
| 정답 입력 UI | 모달 팝업, 문항 수/지선다 수 설정 + 정답 번호 입력 그리드 |
| 자동 채점 | 정답 대조 → 정오 판정 → 점수 계산 |
| ROI Undo | 마지막 박스 1개 되돌리기 (Ctrl+Z) |
| 캔버스 줌 | 확대/축소/화면맞춤 컨트롤 + 마우스 휠 지원 |
| 키보드 단축키 | 모드 전환, 분석 실행, Undo 등 |
| 일괄 처리 | 전체 이미지 연속 분석/채점 + 프로그레스 바 |
| 결과 내보내기 | CSV / Excel 다운로드 |
| 오답 시각화 | 캔버스에서 정답=파란원, 오답=빨간원으로 구분 표시 |

## 6. 제약 사항

- PC 모던 브라우저 필수 (Chrome, Edge 권장)
- 이미지 해상도가 너무 낮으면 판독 정확도 저하
- OMR 버블이 규격화된 양식이어야 정확한 행/열 구조 분석 가능
- 모든 버블이 동일하게 칠해지거나 완전히 비어있는 경우 판별 불가
