# 세션 관리 시스템 기획서

## 개요
OMR 채점 작업을 "세션" 단위로 관리. 1회 모의고사, 2회 모의고사 등을 독립적으로 저장/로드/전환.

---

## 핵심 개념

### 세션 (Session)
- **단위**: 하나의 시험 (예: "2026년 1회 모의고사")
- **포함 데이터**: 과목관리(정답/배점) + 이미지 + 영역설정 + 분석결과 + 채점 + (향후) 성적분석
- **저장 형식**: JSON 파일
- **저장 위치**: 세션 전용 폴더

### 양식 (Template)
- **단위**: OMR 레이아웃 (예: "해커스 OMR 양식")
- **포함 데이터**: 영역 좌표 + 설정 (과목/정답 미포함)
- **저장 형식**: JSON 파일
- **저장 위치**: 양식 전용 폴더

---

## 폴더 구조

```
OMR채점/
├── 세션/                          ← 세션 전용 폴더
│   ├── 2026년 1회 모의고사.json
│   ├── 2026년 2회 모의고사.json
│   └── 중간고사/                   ← 사용자 정의 하위 폴더 (그룹화)
│       ├── 1학년 중간고사.json
│       └── 2학년 중간고사.json
│
└── 양식/                          ← 양식 전용 폴더
    ├── 해커스 OMR.json
    ├── 공무원 OMR.json
    └── 로스쿨/                    ← 하위 폴더
        ├── 언어이해.json
        └── 추리논증.json
```

## 세션 JSON 구조

```javascript
{
    // 메타데이터
    sessionName: "2026년 1회 모의고사",
    createdAt: "2026-04-13T10:00:00",
    lastUsedAt: "2026-04-13T15:30:00",
    version: 1,

    // 과목 관리 (전체)
    subjects: [
        {
            code: "01", name: "국어", numQuestions: 25,
            numChoices: 5, scorePerQuestion: 4, totalScore: 100,
            useCustomScore: false, scoreMap: [],
            answers: "1,2,3,4,5,..."
        },
        ...
    ],

    // 이미지 목록
    images: [
        {
            filename: "학생01.jpg",
            imageData: "data:image/jpeg;base64,...",  // 이미지 데이터 (base64)
            rois: [ ... ],           // 영역 설정
            results: [ ... ],        // 분석 결과
            gradeResult: { ... },    // 채점 결과
        },
        ...
    ],

    // 전역 설정
    settings: {
        intensity: 250,
        bubbleSize: 0,
        // ...
    }
}
```

## 동작 흐름

### 새 세션 시작
1. "새 세션 만들기" 클릭
2. 세션 이름 입력 (예: "2026년 2회 모의고사")
3. 현재 작업 내용 초기화 (or 저장 여부 확인)
4. 빈 상태에서 시작 (과목관리 비어있음, 이미지 없음)

### 세션 저장
1. "현재 세션 저장" 클릭
2. 세션명으로 JSON 파일 생성
3. 세션 폴더에 저장
4. lastUsedAt 갱신
5. **사용자가 명시적으로 저장해야만 저장됨** (자동 저장 안 함)

### 세션 불러오기
1. 세션 목록에서 선택
2. 저장 안 한 현재 작업이 있으면 경고 ("저장하지 않은 변경사항이 있습니다")
3. JSON 로드 → 과목관리, 이미지, 결과 모두 복원
4. 세션 제목 갱신

### 세션 전환
- 불러오기와 동일 (현재 세션 저장 여부 확인 후 전환)

## 파일 시스템 접근 방식

### 방법 1: File System Access API (Chrome/Edge)
- `showDirectoryPicker()`로 세션/양식 폴더 선택 (최초 1회)
- 이후 해당 폴더 내에서 자유롭게 읽기/쓰기
- 하위 폴더 생성/탐색 가능
- 장점: 가장 이상적, 실제 파일 관리
- 단점: Chrome/Edge만 지원, 권한 요청 필요

### 방법 2: 다운로드/업로드
- 저장: JSON 파일 다운로드 (사용자가 폴더 지정)
- 불러오기: 파일 선택 대화상자
- 장점: 모든 브라우저 지원
- 단점: 폴더 자동 탐색 불가, 매번 파일 선택 필요

### 방법 3: React-Electron (확정 배포 방식)
- Electron으로 데스크톱 앱 배포 → 네이티브 파일시스템 접근
- Chrome/Edge/Naver Whale 엔진 기반
- 프로그램 폴더 내에 세션별 하위 폴더 자동 생성
  - `[프로그램폴더]/sessions/[세션명]/` → 이미지 + 세션 JSON
  - `[프로그램폴더]/templates/` → 양식 JSON
- 사용자 별도 액션 없이 자동 정리

### 방법 4: IndexedDB + 시뮬레이션 (웹용 폴백)
- 브라우저 내부 DB에 세션 데이터 저장
- 폴더 구조를 가상으로 시뮬레이션
- 장점: 빠름, 자동 관리
- 단점: 브라우저 데이터 삭제 시 소실, 외부 공유 어려움

### 권장: 방법 1 우선 + 방법 2 폴백
- Chrome/Edge: File System Access API로 네이티브 폴더 관리
- 그 외: 다운로드/업로드 방식

## 이미지 저장 방식

### 문제: 이미지가 크면 JSON 파일이 거대해짐
- 300장 × 2MB = 600MB JSON → 비현실적

### 해결 방안
1. **이미지는 별도 폴더에 원본 유지** → JSON에는 파일 경로만 저장
   - 세션 폴더 안에 `images/` 하위 폴더
   - JSON: `{ filename: "학생01.jpg", path: "images/학생01.jpg" }`
   - 장점: JSON 크기 최소, 이미지 원본 보존
   - 단점: 폴더째 이동해야 함

2. **이미지 압축 저장** → JSON에 축소된 base64
   - 원본 대신 분석에 필요한 해상도로 축소 (예: 1000px 폭)
   - 장점: 단일 파일로 이동 가능
   - 단점: 원본 품질 손실

3. **하이브리드** → 이미지는 별도, 결과만 JSON
   - JSON: 과목 + ROI 설정 + 채점 결과 (이미지 없음)
   - 이미지는 사용자가 같은 폴더에 보관
   - 불러올 때 이미지 폴더 재지정
   - 장점: 유연함
   - 단점: 이미지 분리 시 재연결 필요

## 세션 vs 양식 구분

| 구분 | 세션 | 양식 |
|------|------|------|
| 포함 | 과목+이미지+결과+채점 전부 | 영역 좌표+설정만 |
| 용도 | 특정 시험 하나의 전체 작업 | OMR 레이아웃 재사용 |
| 크기 | 큼 (이미지 포함 시) | 작음 (설정만) |
| 예시 | "2026년 1회 모의고사" | "해커스 OMR 양식" |

---

## 구현 시 주의사항

1. 세션 전환 시 현재 상태 완전 초기화 (과목관리, 이미지, 결과 등)
2. 저장하지 않은 변경 감지 → 전환/종료 시 경고
3. 이미지 데이터 크기 관리 (base64 vs 파일 참조)
4. 파일명에 특수문자 처리
5. 세션 파일 버전 관리 (향후 필드 추가 시 하위 호환)
