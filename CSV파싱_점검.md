# CSV 파싱 로직 전체 점검

## 현재 버그 목록

### 버그 1: 시험인원 CSV — 모든 데이터가 이름 셀 하나에 들어감
**원인 추정**: 쉼표 구분이 안 되고 있음. 
- 엑셀에서 CSV 저장 시 구분자가 탭(\t)이거나 세미콜론(;)일 수 있음
- 또는 _parseCSVLine이 제대로 호출 안 됨
- 또는 파싱은 되는데 cells[0]에 전체가 들어감

**확인 필요**: 실제 CSV 파일의 구분자가 뭔지 (쉼표 vs 탭 vs 세미콜론)

### 버그 2: 과목/정답 CSV — 21행부터 입력하라고 자꾸 뜸
**원인 추정**: `allLines.slice(20)` 후 필터링에서 데이터 행이 걸러짐
- 첫 셀이 숫자가 아니고 빈칸도 아닌 경우 (한글 과목명으로 시작)
- 또는 파일이 20행 미만인데 fallback 로직도 실패

### 버그 3: 코드 없을 때 열 순서 설명 제거 필요
**상태**: CSV 양식에 '[열 순서] 과목코드 없을 때' 설명이 아직 남아있음
- 코드 없으면 `-` 쓰기로 결정됐으므로 별도 열 순서 설명 불필요

## CSV 파싱이 해야 할 것 (정리)

### 과목/정답 CSV

**입력 형식**:
```
코드,과목명,선택지수,배점,총점,1번답,2번답,...,N번답
```
- 코드 없으면 `-` 입력
- 배점이 "차등"이면 정답 뒤에 배점이 이어짐
- 1~20행은 무시, 21행부터 데이터

**파싱 로직**:
1. 파일 읽기 (인코딩: UTF-8 → EUC-KR 폴백)
2. 전체 행 split
3. 21행(index 20)부터 읽기
4. 빈 행 스킵
5. 각 행을 쉼표로 split (큰따옴표 처리)
6. cells[0] = 코드 (`-`이면 빈 코드), cells[1] = 과목명, ...
7. 과목 객체 생성 후 subjects 배열에 추가

**주의**: 
- 탭/세미콜론 구분 파일 대응 필요
- 20행 이하 파일은 하위호환 (전체에서 데이터 탐색)

### 시험인원 CSV

**입력 형식**:
```
이름,생년월일,수험번호,핸드폰번호
```
- 일부 열 비어있어도 됨
- 1~20행은 무시, 21행부터 데이터

**파싱 로직**:
1. 파일 읽기 (인코딩: UTF-8 → EUC-KR 폴백)
2. 전체 행 split
3. 21행(index 20)부터 읽기
4. 빈 행 스킵
5. 각 행을 쉼표로 split
6. cells[0] = 이름, cells[1] = 생년월일, cells[2] = 수험번호, cells[3] = 핸드폰
7. 생년월일/핸드폰 앞자리 0 보존 (_normalizeField)

**현재 버그**: 쉼표가 아닌 구분자(탭) 사용 시 split이 안 됨 → 전체가 cells[0]에 들어감

## 수정 계획

### 1. 구분자 자동 감지
```javascript
_detectDelimiter(line) {
    // 쉼표, 탭, 세미콜론 중 가장 많이 나오는 것
    const commas = (line.match(/,/g) || []).length;
    const tabs = (line.match(/\t/g) || []).length;
    const semis = (line.match(/;/g) || []).length;
    if (tabs > commas && tabs > semis) return '\t';
    if (semis > commas && semis > tabs) return ';';
    return ',';
}
```

### 2. _parseCSVLine에 구분자 파라미터 추가

### 3. 과목 CSV 양식에서 '코드 없을 때 열 순서' 설명 제거
- `-`로 통일됐으므로 별도 설명 불필요

### 4. 21행부터 데이터 읽기 로직 통일
- slice(20) 후 빈 행만 스킵 (쉼표 유무 체크 제거)
- 하위호환: 20행 이하 파일은 전체에서 숫자/한글로 시작하는 쉼표 행 탐색
