## 메커니즘 분석

```
초기 상태:           imgObj.intensity = 115  (사용자 슬라이더)

batch 1회 실행:
  failed 이미지 재시도 → tryIntensity = 115 + 100 = 215
  성공 시 imgObj.intensity = 215   ← 영구 저장

batch 2회 실행 (반복 채점 시):
  failed 다시 → tryIntensity = 215 + 100 = 315
  성공 시 imgObj.intensity = 315

batch 3회: imgObj.intensity = 315 + 100 = 415
batch 4회: 515
...
batch N회: 15 + 100*N
```

여기에 1.5배 검증도 +100% 추가하면 누적 더 빠름.

→ 사용자가 batch를 몇 번 반복하면 진하기가 800%+까지 부풀려짐.

## 영향 범위

### 수정으로 잃는 것 (의도된 동작이었지만 부작용 더 큼)
- 슬라이더가 재시도 성공 후 부풀린 값을 표시 안 함 → 사용자가 "이 이미지는 215%로 인식됨"을 직접 모르게 됨

### 수정으로 얻는 것
- 누적 방지
- 다음 batch 실행 시 사용자 설정값 그대로 baseline으로 사용

### UX 보완 (필요 시 향후)
- 재시도 정보를 별도 필드로 기록: `imgObj._retryUsedIntensity`, `imgObj._retryRotation`
- UI에서 "재시도 인식됨" 뱃지로 표시 — base intensity는 안 건드림

## 기존 데이터 복구

이미 800%까지 누적된 이미지는 사용자가 슬라이더로 직접 100%(또는 원하는 값)로 리셋하거나, "진하기 영역의 [초기화]" 버튼으로 리셋 가능.

`canvas.js:649`의 reset 핸들러:
```js
(App.state.images || []).forEach(imgObj => { imgObj.intensity = 100; });
```
