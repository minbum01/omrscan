# 과목 합치기 (SubjectMerge) 기획 (2026-04-27)

> **상태**: 구현 완료
> **신규 모듈**: `js/subjectMerge.js`

---

## 배경

학생들이 과목 OMR 영역에 잘못 마킹하는 케이스가 자주 발생.
예: A과목 답을 B과목에 마킹, 또는 일부 학생은 A에, 일부는 B에 마킹.

기존엔 동일 ROI에 마킹된 답만 채점되므로 "한 과목이 빠진" 학생이 누락됨.

→ 사용자가 명시적으로 **A를 B에 합침** 등으로 지정해서, 어느 영역에 마킹했든 같은 과목으로 채점되게.

---

## 요구사항 (사용자 합의 사항)

| 케이스 | 결정 |
|---|---|
| 1. 정상 분포 (학생별 다른 영역 마킹) | 각 학생의 마킹된 답을 사용 |
| 2. 같은 학생이 둘 다 마킹 | 적용 시 얼럿 1번 + 그래도 진행 시 **타깃 답** 사용, 소스 답 무시 |
| 3. 정답키 차이 | **타깃의 정답키** 강제 사용 |
| 4. 문항 수/번호 차이 | **합치기 거부** (같은 startNum + 같은 numQuestions만 허용) |
| 5. 다대일 | 가능 (체크박스 여러 개 → 타깃 1개) |
| 6. 표시 | 화면·엑셀 모두 타깃만 노출, 소스 컬럼 숨김 |
| 7. 적용 | 전체 이미지 일괄 + 세션 저장 |
| 8. 되돌리기 | 모달 재오픈 → 그룹 삭제로 복원 (원본 ROI 데이터 무손상) |
| 9. 후보 | `subject_answer` 타입만 |
| 10. 동일 ROI 이름 | 기존 1~20→21~40 concat 로직 그대로 (이 기능과 별개) |

## 데이터 구조

```js
// App.state.subjectMerges
[
  { target: "수학B", sources: ["수학A", "수학C"] },
  // 한 과목명은 한 그룹에만 (source/target 중복 불가)
]
```

저장 위치: 세션 파일의 `subjectMerges` 필드 (`session.js`).

`js/subjectMerge.js`:

| 함수 | 역할 |
|---|---|

| `openModal()`, `_renderModal()`, `_apply()` | 모달 UI |

## 채점 로직 핵심 (`_applyGroup`)

각 학생 row에 대해:

1. **byQ 초기화** — 타깃의 marked만 우선 채움 (채점값 isCorrect/correctAnswer는 무시)
2. **소스에서 marked 채움** — 타깃이 비어있을 때만 (타깃 우선)
3. **타깃 정답키로 일괄 재채점** — 모든 q에 대해 `info.keyByQ[q]`로 isCorrect 재계산
4. **점수 합산** — totalPossible/score/correctCount/wrongCount 재계산
5. **소스 제거** — `delete row.subjects[s]`

```
케이스별 동작:
- B만 마킹 → B의 마킹, B 정답키로 채점
- A만 마킹 → A의 마킹, B 정답키로 재채점
- A·B 모두 마킹 → B의 마킹 (A 무시), B 정답키로 채점
- 둘 다 빈칸 → null, 빈칸 처리
```

## UI 진입점

**위치**: 분석탭 우측 패널 → "영역 목록" 버튼 펼침 → 목록 상단 "🔗 과목 합치기" 버튼

**모달 구조**:
- 그룹 카드 (여러 개 가능)
- 각 그룹: 타깃(라디오, 1개) + 소스(체크박스, 여러 개)
- 다른 그룹에 사용된 과목은 disabled 회색 표시
- "+ 새 그룹" / "취소" / "적용"
- 검증 실패 시 빨간 경고 (그룹 내 표시)
- 적용 시 충돌 감지 → confirm 얼럿 (학생 명단 / 건수)

## 영향받는 함수들

| 함수 | 영향 |
|---|---|
| `Scoring.collectData()` | 끝에서 `SubjectMerge.applyToRows(result)` 호출 — 자동 적용 |
| `Scoring.calcStats / calcItemAnalysis / getSubjectList` | 변환된 rows를 받음 — 자동 반영 |
| `export.js` | `Scoring.collectData()`/`getSubjectList`를 사용 — 엑셀에도 타깃만 노출 |
| `publicReport.js` | 동일 — 게시용 성적표에도 자동 반영 |

→ 비파괴 후처리이므로 **수정한 함수 외엔 모두 자동으로 변환된 데이터를 받음**.

## 세션 저장/복원

`session.js`:
- 저장 시 `subjectMerges: App.state.subjectMerges || []` 추가
- 복원 시 `App.state.subjectMerges = Array.isArray(data.subjectMerges) ? data.subjectMerges : []`
- 새 세션 시 `App.state.subjectMerges = []` 초기화

## 향후 확장 가능성

- 합치기 그룹 자동 추천 (이름 유사도 분석)
- 그룹별 채점 통계 (몇 명이 소스 vs 타깃에 마킹했는지)
- 마스터 DB 도입 시 학원 차원에서 표준 과목명 매핑과 통합
