# 게시용 성적표 v3 (2026-04-27)

> **상태**: 구현 완료
> **파일**: `js/publicReport.js` (전면 재작성)
> **출력**: 정적 HTML, A3 portrait (297mm × 420mm)

---

## 디자인 원칙

- **A3 strict size**: 297×420mm 고정, 초과 시 잘림 → 사용자가 데이터 누락 인지 가능
- **portrait orientation**: 세로 길게
- **인쇄 친화**: `@media print`로 설정 패널·탭바·리사이즈 핸들 숨김
- **사용자 커스터마이징**: 거의 모든 시각 요소를 인라인 편집·설정 패널로 조정

## 구조

### 탭
1. **성적 리포트** — 배너 + 점수 분포 차트 + 문항분석
2. **학생 성적표** — 배너 + 학생 테이블

각 탭은 자체 paper(297×420mm) 안에 콘텐츠. 우측에 설정 패널.

### 배너 (banner)
- 그라데이션 배경 (시작/끝 색상 조정 가능)
- 워드마크 ("HACKERS" 등) — paper 상단 큰 글씨, contenteditable
- 워터마크 — 배너 우측 거대 텍스트, 반투명, contenteditable
- 제목·부제·날짜 — contenteditable
- KPI 4개: 응시인원 / 평균점수 / 평균맞은수 / 최고점

```
generate()
```

### reportData 확장 필드

- `etcKeys` — 모든 학생의 etcFields 키 합집합

## 점수 분포 차트

### 구간 모드 (calcBins)
- **자동**: `[0,0]` 단독 + `[1, bin]`, `[bin+1, 2*bin]`, ..., 마지막 bin은 max 포함 (옵션 B)
- **수동**: 사용자가 상한값을 쉼표로 입력 (예: `5,10,20,40,100`)

### 표시 구간 토글 (칩)
- 모든 구간을 칩으로 표시 (활성=파란 bg, 비활성=회색+취소선)
- 클릭으로 토글 → 표/차트에서 즉시 반영
- "전체 표시" / "전체 숨김" 버튼
- 위치: 우측 설정 패널 "점수 분포 구간" 그룹 안

### 차트 그리기 (drawDist)
- DPR 3배 buffer (canvas.width = W * 3)
- setTransform(3, 0, 0, 3, 0, 0)으로 논리 좌표계 유지
- 그라데이션 막대, 그림자, 하이라이트, 데이터 라벨, 그리드

## 문항분석

### 테이블 (`.tbl-compact`)
- Q / 정답 / 정답률 / 상위% / 하위% / 변별도 / ①②③④⑤
- 상위%/하위%/변별도 토글 (디폴트 변별도 OFF)
- ①~⑤ 컬럼 너비 통일 (`.c-ch` 44px, `white-space:nowrap`)
- 헤더 타이포 키움 (12px, weight 700)

### 차트 (drawItem)
- 가로 막대 차트 (각 문항 정답률)
- 평균선(주황 dash) + 상위% 평균선(녹색 dash)
- 색상: 정답률 < 40% 빨강 / > 80% 녹색 / 중간 파랑

## 학생 테이블

### 모드
- **단일 (single)**: 한 테이블, 세로로 길게
- **다단 (multi)**: N×C 청크 배치, 섹션이 아래로 쌓임

### 컬럼 (18가지 토글)
- 인적사항: 수험번호 / 이름 / 전화번호 / 생년월일 / 과목코드
- 기타 필드: etcFields (동적, ROI에 따라)
- 과목별: 점수 / 맞은수 / 틀린수 / 만점 / 석차 / 백분위 / 표준점수
- 종합: 총점 / 총 맞은수 / 총 틀린수 / 총 만점 / 종합석차 / 종합백분위 / 종합표준점수

### 다단 모드 헤더 (β)
- 모든 청크에 헤더 라벨 표시
- 컬럼 너비 드래그 시 같은 `data-col-key`인 모든 th 동기화 → 청크 간 정렬 유지

### 정렬
- `_stuSort` 상태 + `stuSort(colKey)` 함수
- 단일·다단 동일 동작 — `D.students` 정렬 후 재렌더
- # 컬럼은 정렬 대상 제외, 항상 1, 2, 3...로 표시 (display index)
- 활성 정렬 컬럼 헤더에 ↑/↓ 화살표

### '?' 별색 처리
- 셀에 '?' 포함 시 `cell-q` 클래스 추가 → 빨간 배경 (`#fee2e2`)
- 마스킹 시 '?'가 가려지면 `cell-q` 자동 제거 (블러: 항상, 별표: 마스킹 후 텍스트에 '?' 안 남으면)

### 마스킹
- 대상: 전체 / 수험번호 / 이름 / 전화번호 / 생년월일
- 방식: 블러 / 별표(*)
- `_maskActive` 플래그로 상태 추적
- `buildStudent` 끝에서 `reapplyMaskIfActive()` 호출 → 설정 변경 후에도 마스킹 보존

### 컬럼 너비 드래그
- 각 `<th>` 우측 6px 핸들 (`onmousedown="startColResize"`)
- 드래그 시 width/min-width/max-width 갱신
- 다단 모드: 같은 `data-col-key`인 모든 th 동기화

## 차트 리사이즈

### 동작
- `.chart-wrap`에 `resize: both; overflow: hidden`
- 사용자가 우하단 모서리 드래그 → wrap 크기 변경
- ResizeObserver가 감지 → canvas 크기 갱신 + 재그리기

### 핵심 함수
- `_setCanvasFromWrap(wrap, cv, reserveBottom)` — wrap 크기 → canvas 크기 (DPR 3배 적용)
- `_ctxLogical(cv)` — 그리기 시작 헬퍼 (논리 W/H + setTransform)
- `setupChartResize()` — ResizeObserver 등록 (rebuild 끝에 호출)

### DPR 3배 (인쇄 화질)
- 화면 표시: cv.style.width = W (CSS 픽셀)
- 실제 buffer: cv.width = W × 3 (해상도)
- 그리기: setTransform(3,0,0,3,0,0) → 논리 좌표계 W×H로 그리되 buffer는 3x
- 인쇄 시 buffer를 raster → 픽셀 깨짐 거의 없음

## 워드마크 / 헤더 사용자 조정

### 워드마크 (상단 큰 텍스트)
- 텍스트: 클릭으로 직접 편집
- 크기 (px) / 색상 / 그라데이션 ON·OFF

### 워터마크 (배너 우측 거대 텍스트)
- 텍스트: 클릭으로 직접 편집
- 표시 ON·OFF / 크기 / 진하기 (%)
- 디폴트: 80px, 8% 투명도

### 배너 헤더
- 제목 / 부제 / KPI 값 / KPI 라벨 — 폰트 크기 조정
- 배경 그라데이션 시작·끝 색상 조정
