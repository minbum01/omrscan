## 1. 분석탭 UX 개선 — 전체 빈칸 처리 워크플로우
**파일**: `js/ui.js`

- `markAllBlank(roiIdx)` (ui.js:2245) 후 현재 이미지에 더 처리할 ROI 없으면 `moveToNextImageError()` 호출 → 다음 이미지 자동 이동
- `_roiNeedsCorrection(res)` 헬퍼 추가 (ui.js:2238) — 미처리 오류 행 있는지 판정
- `renderCombinedPanel`의 ROI 카드 순회를 정렬 인덱스로 변경 → 교정 필요 ROI 최상단에
- `moveToNextImageError`의 hasError 체크에 `!r.corrected` 추가 — 이미 처리된 빈칸 제외
- `undetected = false`도 `markAllBlank`에서 추가 리셋

**효과**: 사용자가 마우스를 한 자리에 두고 클릭 반복 → 자동으로 다음 처리 대상이 그 자리에 오름.

## 2. 과목 합치기 (SubjectMerge) — 신규 기능
**신규 파일**: `js/subjectMerge.js`
**상세**: [`0427_과목합치기_기획.md`](./0427_과목합치기_기획.md)

- 학생별로 마킹 영역이 다른 경우(A 또는 B) 한 과목으로 통합 채점
- 비파괴: 원본 ROI/results 안 건드리고 `App.state.subjectMerges`에 매핑만 저장
- `Scoring.collectData()` 끝에서 `SubjectMerge.applyToRows(rows)` 적용
- 분석탭 "영역 목록" 펼침 패널 상단에 "🔗 과목 합치기" 버튼 + 모달
- 검증: startNum + numQuestions 일치 필수 (불일치 시 거부)
- 충돌(둘 다 마킹) 시 confirm 얼럿 + 그래도 진행 시 타깃 답 사용
- 세션 저장/복원 (`session.js`에 `subjectMerges` 필드)

**핵심 수정**: `_applyGroup`에서 모든 행을 타깃 정답키로 일괄 재채점 (sub.answers 갱신, isCorrect 재계산).

## 3. 게시용 성적표 v3 (전면 재설계)
**파일**: `js/publicReport.js` (전면 재작성)
**상세**: [`0427_게시용성적표_v3_기획.md`](./0427_게시용성적표_v3_기획.md)

### 구조
- 탭 구조: 성적 리포트 / 학생 성적표
- A3 strict 사이즈 (297mm × 420mm, overflow:hidden)
- 우측 설정 패널 (collapsible 그룹들)

### 기능
- **점수 분포 차트**: 자동/수동 모드, 0 단독 + 균등 구간(자동), 사용자 정의 상한값(수동)
- **표시 구간 칩 토글**: 체크박스 → 칩(클릭으로 ON/OFF, 비활성=취소선)
- **학생 테이블**: 단일/다단(N×C) 모드, β 헤더(모든 청크에 라벨), 컬럼 너비 드래그 (data-col-key 동기화)
- **컬럼 토글**: 18가지 옵션 (수험번호/이름/전화번호/생년월일/과목코드/etc/과목별 점수·맞은수·틀린수·만점·석차·백분위·표준점수/종합 7종)
- **마스킹**: 전체/수험번호/이름/전화번호/생년월일 — 설정 변경 후에도 보존 (`reapplyMaskIfActive`)
- **'?' 별색 처리**: 셀에 '?' 포함 시 빨간 배경 (`cell-q`) — 마스킹되면 자동 해제
- **차트 리사이즈**: `.chart-wrap`에 `resize:both` + ResizeObserver 그리기 갱신
- **차트 화질**: DPR 3배 (canvas buffer 3x, CSS 1x) → 인쇄 시 선명
- **워드마크 / 헤더**: 텍스트 직접 편집 + 크기/색상/배경 조정 (설정 패널)

### 디폴트
- 점수 분포: 표 20px, 축 15px, 제목 22px
- 변별도: OFF
- 학생 테이블: 다단, 이름/총점/석차/백분위 OFF
- 헤더 KPI: 응시인원/평균/평균맞은수/최고점 (4개, 표준편차 제거)
- 워드마크: "HACKERS" 56px 그라데이션
- 워터마크: 배너 우측 80px / 8% 투명도
